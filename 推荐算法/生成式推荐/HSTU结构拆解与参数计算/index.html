HSTU结构拆解与参数计算

发表于2026-06-13|更新于2026-06-13|推荐算法生成式推荐

HSTU 是 Meta 在 2024 年提出的推荐序列建模结构，全称是 Hierarchical Sequential Transduction Unit。它的核心思路是把推荐系统里的异构特征统一成时间序列，然后用堆叠的 HSTU block 对序列做表示学习。HSTU 在推荐系统里承担 sequence encoder 的角色，输出一串序列表示。本文只讨论 HSTU 结构本身：输入序列如何构造，单层 HSTU 如何计算，输出张量是什么形状，以及在一个典型配置下如何估算模型参数量。

1. HSTU 在推荐模型中的位置

HSTU，全称 Hierarchical Sequential Transduction Unit，出现在 Meta 的 generative recommendation 工作中。完整的 Generative Recommender 并不只有 HSTU：前面要把用户行为、item、action、时间和上下文特征整理成序列化输入，后面还会接 retrieval、ranking 或生成式训练目标。HSTU 模块承担 sequence encoder 的角色。

传统 DLRM 更像三段式结构：先做 feature extraction，再做 feature interaction，最后做 representation transformation。HSTU-based GR 的思路不同，它把推荐特征统一整理成时间序列，然后用堆叠的 HSTU block 对序列做表示学习。

左侧是传统 DLRM 的三段式结构 vs 右侧是简化后的 HSTU 主干。HSTU 的核心差异点：

  • 第一，HSTU 的输入不是普通表格特征拼接，而是 Sequentialized Unified Features。也就是先把异构推荐特征统一成按时间排列的 token 序列。

  • 第二，HSTU 不是简单在 DLRM 后面加一个 Transformer。它试图把推荐系统里原本分散在特征交互网络和上层表示变换网络中的一部分能力，收进一个序列建模主干里。每一层 HSTU 都会重复做三件事：投影、序列聚合、门控变换。

图1：论文 Figure 3，传统 DLRM 与 HSTU-based Generative Recommender 的结构对比。

论文中一层 HSTU 主要由三块组成：

  1. Pointwise Projection：对每个位置的 token 做投影，得到后续需要的 U,Q,K,VU,Q,K,VU,Q,K,V。
  2. Spatial Aggregation：用 QKTQK^TQKT 加相对位置、相对时间偏置，聚合序列信息。
  3. Pointwise Transformation：把聚合结果和门控信号结合后，再投影回模型维度。

本文重点围绕这三块展开，不讨论完整工业推荐系统中的召回链路、样本生成、训练数据流和在线服务细节。

2. 输入 X 是怎么构造的

HSTU 的输入可以写成：

X∈Rn×dX \in \mathbb{R}^{n \times d} X∈Rn×d

这里 nnn 是序列长度，ddd 是每个 token 的表示维度。比如 n=8192,d=1024n = 8192, d = 1024n=8192,d=1024。也就是一条序列样本有 8192 个 token，每个 token 最终都会被编码成一个 1024 维向量。

HSTU 论文里强调输入序列不是“只把 item_id 串起来”，而是把 DLRM 里的异构特征尽量转成统一时间序列。

图2：论文 Figure 2，DLRM 特征到 Generative Recommender 序列化特征的转换方式。

可以把输入构造分成三类信息。

第一类是主行为序列。
这通常来自用户和内容的交互历史，例如浏览、点击、点赞、收藏、评论、购买等。每个被交互的内容可以记为 Φi\Phi_iΦi​，对应的动作记为 aia_iai​，发生时间记为 tit_iti​。一条用户历史可以抽象成：

(Φ0,a0,t0),(Φ1,a1,t1),…,(Φm−1,am−1,tm−1)(\Phi_0, a_0, t_0),(\Phi_1, a_1, t_1),\ldots,(\Phi_{m-1}, a_{m-1}, t_{m-1}) (Φ0​,a0​,t0​),(Φ1​,a1​,t1​),…,(Φm−1​,am−1​,tm−1​)

其中 Φi\Phi_iΦi​ 往往是 item_id、content_id 这类高基数 ID（数量很大，百万/千万/上亿），它们通过 embedding table 查表得到向量。

第二类是动作和时间特征。
动作 aia_iai​ 可以表示 click、like、share、purchase、skip 等行为类型。它通常也是一个离散特征，可以通过 action embedding 编码。时间 tit_iti​ 可以有多种编码方式：绝对时间、相对时间间隔、时间桶、session 内位置、距离当前请求的 elapsed time 等。HSTU 在 attention aggregation 中还会使用相对位置和相对时间偏置 rabp,trab^{p,t}rabp,t，让模型显式感知“顺序距离”和“时间间隔”。

第三类是慢变化或上下文类别特征。
例如用户语言、地区、设备、长期兴趣、关注的 creator、加入的 community、请求场景等。这类特征不一定每次行为都变化，论文里的做法是把它们压缩后合并进主时间线。直观理解是：主行为序列提供时间骨架，其他类别特征在合适的位置插入或对齐，最后变成同一种 token 序列。

实际进入 HSTU 的 xix_ixi​ 往往不是单一 embedding，而是多种编码后的融合结果。可以写成一个抽象形式：

xi=Combine⁡(eitem(Φi),eaction(ai),etime(ti),econtext(ci))x_i = \operatorname{Combine}\left( e_{\text{item}}(\Phi_i), e_{\text{action}}(a_i), e_{\text{time}}(t_i), e_{\text{context}}(c_i) \right) xi​=Combine(eitem​(Φi​),eaction​(ai​),etime​(ti​),econtext​(ci​))

其中 Combine 可以是相加、拼接后投影、或者更复杂的 embedding operator。无论前面怎么组织，进入 HSTU block 后的形状统一为：

X=[x1,x2,…,xn],xi∈RdX = [x_1,x_2,\ldots,x_n],\quad x_i \in \mathbb{R}^{d} X=[x1​,x2​,…,xn​],xi​∈Rd

3. 单层 HSTU 的模型结构

单层 HSTU 的输入是 X∈Rn×dX \in \mathbb{R}^{n \times d}X∈Rn×d，输出仍然是 Y∈Rn×dY \in \mathbb{R}^{n \times d}Y∈Rn×d。单层 HSTU block 流程：每层先产生 U,Q,K,VU,Q,K,VU,Q,K,V，再做带相对偏置的序列聚合，最后通过 Norm、门控和输出投影得到下一层表示。

图3：论文 Figure 3 右侧的单层 HSTU block 局部结构。

3.1 Pointwise Projection：从 X 到 U、Q、K、V

第一步是 pointwise projection。它对每个序列位置使用共享的 MLP 或线性投影，把输入 XXX 的每个 token 从 ddd 维升到 4d4d4d 维，然后沿特征维直接切分成四组 ddd 维向量：

U(X),Q(X),K(X),V(X)=Split⁡(ϕ1(f1(X)))U(X), Q(X), K(X), V(X) = \operatorname{Split}(\phi_1(f_1(X))) U(X),Q(X),K(X),V(X)=Split(ϕ1​(f1​(X)))

更明确地写，就是：

Z=ϕ1(f1(X)),Z∈Rn×4dZ = \phi_1(f_1(X)), \quad Z \in \mathbb{R}^{n \times 4d} Z=ϕ1​(f1​(X)),Z∈Rn×4d

U,Q,K,V=split⁡(Z),U,Q,K,V∈Rn×dU,Q,K,V = \operatorname{split}(Z), \quad U,Q,K,V \in \mathbb{R}^{n \times d} U,Q,K,V=split(Z),U,Q,K,V∈Rn×d

这里的 pointwise 指的是“逐位置处理”：同一个投影函数作用在每个 token 上，先不直接混合不同位置的信息。

Q,K,VQ,K,VQ,K,V 可以沿用 attention 的直觉来理解：QQQ 和 KKK 用来计算位置之间的相关性，VVV 是被聚合的信息。UUU 是门控分支，后面会控制哪些维度应该通过。

3.2 Spatial Aggregation：用相对位置和时间聚合历史

第二步是 spatial aggregation。上一小节已经得到每个位置的 Q,K,VQ,K,VQ,K,V：

Q=[Q1,Q2,…,Qn],K=[K1,K2,…,Kn],V=[V1,V2,…,Vn]Q = [Q_1,Q_2,\ldots,Q_n],\quad K = [K_1,K_2,\ldots,K_n],\quad V = [V_1,V_2,\ldots,V_n] Q=[Q1​,Q2​,…,Qn​],K=[K1​,K2​,…,Kn​],V=[V1​,V2​,…,Vn​]

其中 Qi,Ki,Vi∈RdQ_i,K_i,V_i \in \mathbb{R}^{d}Qi​,Ki​,Vi​∈Rd。

对第 iii 个位置来说，它会和第 jjj 个位置计算一个相关性分数：

sij=QiKjT+rabijp,ts_{ij} = Q_i K_j^T + rab^{p,t}_{ij} sij​=Qi​KjT​+rabijp,t​

其中，rabi,jp,trab^{p,t}_{i,j}rabi,jp,t​ 是 relative attention bias，表示相对位置和相对时间偏置。对固定的 token pair (i,j)(i,j)(i,j) 来说，它是一个标量 bias；但它不是全局常数，不同位置对会有不同的 bias 值。论文没有给出完整闭式公式，但 Table 9 说明可以对 token pair (i,j)(i,j)(i,j) 的时间差 tj−tit_j-t_itj​−ti​ 做 bucketization。一个可能的实现方式是：

rabijp,t=bbucket⁡(j−i)p+bbucket⁡(tj−ti)trab^{p,t}_{ij} = b^p_{\operatorname{bucket}(j-i)} + b^t_{\operatorname{bucket}(t_j-t_i)} rabijp,t​=bbucket(j−i)p​+bbucket(tj​−ti​)t​

这里 bpb^pbp 是可学习的相对位置 bias 表，btb^tbt 是可学习的相对时间 bias 表，分桶（bucketization）负责把连续或大范围的距离压到有限个桶里，例如 1,2,3,4-7,8-15,16-31,31+1,2,3,4\text{-}7,8\text{-}15,16\text{-}31,31+1,2,3,4-7,8-15,16-31,31+。论文还说明，实际中 rabp,trab^{p,t}rabp,t 可以在同一层内跨 attention heads 共享。

把所有 i,ji,ji,j 的相关性分数放在一起，得到 attention score 矩阵：

S=Q(X)K(X)T+rabp,tS = Q(X)K(X)^T + rab^{p,t} S=Q(X)K(X)T+rabp,t

然后再经过 ϕ2\phi_2ϕ2​ 得到 attention 权重矩阵：

A(X)=ϕ2(S)A(X) = \phi_2(S) A(X)=ϕ2​(S)

ϕ2\phi_2ϕ2​ 是非线性函数。论文中 ϕ1\phi_1ϕ1​ 和 ϕ2\phi_2ϕ2​ 都使用 SiLU：

SiLU⁡(z)=z⋅σ(z)\operatorname{SiLU}(z) = z \cdot \sigma(z) SiLU(z)=z⋅σ(z)

所以 HSTU 不是把 QKT+rabp,tQK^T + rab^{p,t}QKT+rabp,t 送进 softmax，而是对每个 pair 的 score 做 pointwise 的 SiLU 激活：

Aij=SiLU⁡(QiKjT+rabijp,t)A_{ij} = \operatorname{SiLU}(Q_iK_j^T + rab^{p,t}_{ij}) Aij​=SiLU(Qi​KjT​+rabijp,t​)

这就是论文里说的 pointwise aggregated attention。标准 Transformer 的 softmax attention 会在一整行上做归一化：

softmax⁡(si1,si2,…,sin)\operatorname{softmax}(s_{i1},s_{i2},\ldots,s_{in}) softmax(si1​,si2​,…,sin​)

因此每一行权重通常会被归一化成和为 1。HSTU 这里没有用 softmax 做行归一化，而是对每个 sijs_{ij}sij​ 独立做 SiLU。论文的动机是：推荐场景里，相关历史行为的数量本身也是强信号；如果用 softmax，很多“强相关行为累积起来的强度”会被归一化掉。

最终得到的 A(X)A(X)A(X) ：

A(X)∈Rn×nA(X) \in \mathbb{R}^{n \times n} A(X)∈Rn×n

图4：HSTU Spatial Aggregation 示意图，展示从 Q/K/V、relative attention bias、SiLU 到 A(X) 的计算路径。

3.3 Gate + Norm：用 U 控制聚合结果

得到 A(X)A(X)A(X) 后，再用它聚合 V(X)V(X)V(X)：

A(X)V(X)A(X)V(X) A(X)V(X)

展开到第 iii 个位置，就是：

(A(X)V(X))i=∑j=1nAijVj(A(X)V(X))_i = \sum_{j=1}^{n} A_{ij}V_j (A(X)V(X))i​=j=1∑n​Aij​Vj​

聚合结果先做归一化，再和 U(X)U(X)U(X) 做逐元素乘法：

Norm⁡(A(X)V(X))⊙U(X)\operatorname{Norm}(A(X)V(X)) \odot U(X) Norm(A(X)V(X))⊙U(X)

这个结构可以看成一种门控。A(X)V(X)A(X)V(X)A(X)V(X) 表示从历史位置聚合来的信息，U(X)U(X)U(X) 决定这些信息在各个维度上如何通过。

3.4 Pointwise Transformation：投影回 d 维

经过 3.2 和 3.3 后，每个位置已经拿到了“聚合后的历史信息”，并且这个信息已经被 U(X)U(X)U(X) 做过门控。把门控后的中间结果记为：

G(X)=Norm⁡(A(X)V(X))⊙U(X)G(X)=\operatorname{Norm}(A(X)V(X)) \odot U(X) G(X)=Norm(A(X)V(X))⊙U(X)

在前面的简化写法里：

G(X)∈Rn×dG(X) \in \mathbb{R}^{n \times d} G(X)∈Rn×d

这里的 G(X)G(X)G(X) 就是 f2f_2f2​ 的输入。也就是说，f2f_2f2​ 接收的不是 A(X)A(X)A(X) 这个 n×nn \times nn×n 的权重矩阵，而是 attention 聚合并门控之后的 token 表示。

最后一步是 pointwise transformation：

Y(X)=f2(G(X))Y(X) = f_2(G(X)) Y(X)=f2​(G(X))

f2f_2f2​ 在论文中表示一个 MLP，实际为了降低计算复杂度，论文采用一层线性层：

f2(G)=GW2+b2f_2(G) = GW_2 + b_2 f2​(G)=GW2​+b2​

这里的 pointwise 仍然表示“逐位置变换”：同一个 f2f_2f2​ 作用在每个 token 的表示上，不再额外做位置之间的混合。位置之间的信息混合已经在 A(X)V(X)A(X)V(X)A(X)V(X) 里完成了，f2f_2f2​ 负责对每个位置的特征维做变换，把门控后的表示整理成下一层需要的输出。

4. 参数量怎么估算

参数估算可以先从单层开始，再乘以层数，最后再看 embedding 层的参数。输入配置取 n=8192,d=1024n = 8192, d = 1024n=8192,d=1024，把单层、24 层 encoder 和 embedding table 放在一起估算。

4.1 HSTU 结构参数量估算

先看单层 HSTU。设模型维度为：

d=1024d = 1024 d=1024

第一块是 f1f_1f1​ projection，把 XXX 投影成 U,Q,K,VU,Q,K,VU,Q,K,V。如果按一个合并矩阵估算，就是：

d→4dd \rightarrow 4d d→4d

参数量为：

d×4d=4d2d \times 4d = 4d^2 d×4d=4d2

代入 d=1024d=1024d=1024：

4d2=4×10242≈4.19M4d^2 = 4 \times 1024^2 \approx 4.19M 4d2=4×10242≈4.19M

第二块是 f2f_2f2​ output projection。先按论文公式主路径来看，f2f_2f2​ 接收的是门控后的结果：

Norm⁡(A(X)V(X))⊙U(X)∈Rn×d\operatorname{Norm}(A(X)V(X)) \odot U(X) \in \mathbb{R}^{n \times d} Norm(A(X)V(X))⊙U(X)∈Rn×d

因此 f2f_2f2​ 可以按：

d→dd \rightarrow d d→d

d×d=d2d \times d = d^2 d×d=d2

24×5.24M≈126M24 \times 5.24M \approx 126M 24×5.24M≈126M

Concat⁡(A(X)V(X),X,U(X))∈Rn×3d\operatorname{Concat}\left(A(X)V(X), X, U(X)\right) \in \mathbb{R}^{n \times 3d} Concat(A(X)V(X),X,U(X))∈Rn×3d

4d2+3d2=7d24d^2 + 3d^2 = 7d^2 4d2+3d2=7d2

5. 总结

HSTU 是 generative recommendation 中的序列编码主干。它接收的不是一组松散表格特征，而是经过 sequentialization 和 embedding 编码后的统一序列：

X∈Rn×dX \in \mathbb{R}^{n \times d} X∈Rn×d

单层计算可以记成三步：先投影出 U,Q,K,VU,Q,K,VU,Q,K,V，再用带相对位置和时间偏置的 attention aggregation 聚合历史信息，最后通过 Norm、门控和输出投影回到 ddd 维。输出仍然是：

Y∈Rn×dY \in \mathbb{R}^{n \times d} Y∈Rn×d

堆叠多层以后，HSTU 得到的是一串更强的用户行为表示。本文到这里为止只讨论 encoder 输出本身，不展开后续召回、排序任务的系统用法。

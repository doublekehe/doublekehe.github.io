推荐系统双塔召回模型详解

发表于2026-04-03|更新于2026-04-03|推荐算法召回

从矩阵分解到深度学习，双塔如何成为现代推荐系统的召回标配

1. 双塔模型的基础介绍

1.1 从矩阵分解说起

在了解双塔模型之前，我们先回顾一下矩阵分解（Matrix Factorization，MF）：

矩阵分解：将用户-物品评分矩阵 R∈Rm×nR \in \mathbb{R}^{m \times n}R∈Rm×n 分解为两个低秩矩阵的乘积：R≈U⋅V⊤R \approx U \cdot V^{\top}R≈U⋅V⊤，通过梯度下降学习用户隐向量 uu∈Rku_u \in \mathbb{R}^kuu​∈Rk 和物品隐向量 vi∈Rkv_i \in \mathbb{R}^kvi​∈Rk，用点积 uu⋅viu_u \cdot v_iuu​⋅vi​ 预测用户uuu对物品iii的偏好分。

矩阵分解的局限：

  • 只用 ID 信息，没有利用用户和物品丰富的内容特征
  • 无法处理新用户、新物品的冷启动问题

1.2 双塔模型的核心思想

双塔模型（Two-Tower Model）的核心洞察是：

用两个深度神经网络（DNN）分别学习矩阵分解中的 user/item ID Embedding，同时融合用户/物品的内容特征，学习更丰富的表征向量，再通过向量检索完成高效召回。

1.2.1 用户侧特征（User Tower）

特征类型 具体特征 说明
ID 类特征 用户ID、设备ID 用户唯一标识
画像特征 性别、年龄、职业、地域、人生阶段 用户人口属性与生命周期
统计特征 30日点击次数、7日活跃天数、平均停留时长、最近活跃距今天数、历史转化次数 用户行为活跃度与质量

1.2.2 物品侧特征（Item Tower）

特征类型 具体特征 说明
ID 类特征 物品ID、类目ID、品牌ID、店铺ID 物品唯一标识与归属
画像特征 类目层级、价格带、品牌档位、上市季节、商品标签（Top20） 商品内容与属性描述
统计特征 7日曝光量、7日点击率、7日转化率、加购次数、好评率 商品历史表现数据

1.3 双塔模型 vs 矩阵分解

对比维度 矩阵分解（MF） 双塔模型
特征 仅用户ID + 物品ID ID + 内容特征（类目/价格/标签等）
用户向量 ID Embedding（线性） DNN 输出的用户表征向量（非线性）
物品向量 ID Embedding（线性） DNN 输出的物品表征向量（非线性）
泛化能力 弱（仅靠ID） 强（融合内容特征，缓解冷启动）

2. 双塔模型的训练方法

双塔模型的训练可以采用pointwise、pairwise、listwise三种方式，下面分别介绍。

2.1 核心区别：如何构建样本 + 如何设计 Loss

三种训练方式的核心区别在于**“如何构建训练样本"以及"损失函数的设计目标”**不同。

简单来说，就是模型在训练时：看一个样本，还是看一对样本，还是看一组样本？

2.2 Pointwise：逐点训练 🎯

核心思想

Pointwise 将召回问题转化为一个二分类问题：模型独立地看待每一个样本。

  • 对于正样本（用户点击/购买过的物品），模型试图让相似度分数接近 1
  • 对于负样本（随机采样的物品），模型试图让相似度分数接近 0

样本构造

单点输入：(用户, 物品, 标签)

样本 用户 物品 标签
正样本 用户A 商品1 1（喜欢）
负样本 用户A 商品3 0（不喜欢）

损失函数：交叉熵损失（Cross-Entropy）

Lpointwise=−∑(u,i,y)∈D[y⋅log⁡σ(uu⋅vi)+(1−y)⋅log⁡(1−σ(uu⋅vi))]L_{\text{pointwise}} = - \sum_{(u, i, y) \in \mathcal{D}} \left[ y \cdot \log \sigma(u_u \cdot v_i) + (1-y) \cdot \log (1 - \sigma(u_u \cdot v_i)) \right] Lpointwise​=−(u,i,y)∈D∑​[y⋅logσ(uu​⋅vi​)+(1−y)⋅log(1−σ(uu​⋅vi​))]

其中 σ(⋅)\sigma(\cdot)σ(⋅) 为 Sigmoid 函数，y=1y=1y=1 为正样本，y=0y=0y=0 为负样本。

特点

优点 缺点
实现简单，训练速度快 忽略物品之间的相对关系
只关心"这个对不对"，不关心"这个是否比那个更好"

💡 工业经验：正负样本比例通常控制在 1:2 ~ 1:3 左右，是一个被广泛验证的经验值。

2.3 Pairwise：成对训练 ⚖️

核心思想

Pairwise 更关注相对排序：让正样本的分数尽量高于负样本。

每次输入一个三元组：(用户, 正样本, 负样本)

模型的目标不是预测具体的分数，而是拉开正负样本之间的距离：

用户-正样本相似度 > 用户-负样本相似度 + 间隔（margin）

样本构造

三元组输入：(用户, 正样本物品, 负样本物品)

字段 内容
用户 用户A
正样本 用户A 点击过的商品1
负样本 用户A 从未点击的商品3

损失函数：Triplet Hinge Loss

Lpairwise=∑(u,i+,i−)max⁡ ⁣(0, γ−σ(uu⋅vi+)+σ(uu⋅vi−))L_{\text{pairwise}} = \sum_{(u, i^+, i^-)} \max\!\left(0,\ \gamma - \sigma(u_u \cdot v_{i^+}) + \sigma(u_u \cdot v_{i^-})\right) Lpairwise​=(u,i+,i−)∑​max(0, γ−σ(uu​⋅vi+​)+σ(uu​⋅vi−​))

其中 γ\gammaγ 为间隔阈值（通常取 0.5~1.0）。

特点

优点 缺点
直接优化排序指标，更符合推荐"把喜欢的排在前面"的目标 训练收敛相对较慢
效果稳健 对负样本的选择比较敏感

2.4 Listwise：列表训练 📋（工业主流）

核心思想

Listwise 是目前大厂（如 YouTube）在双塔模型中最常用的方式，效果通常最好。

每次输入一组数据：(用户, 1个正样本, N个负样本)，被看作一个多分类问题（Multi-class Classification）。

模型的目标是：让正样本在这一组候选中的预测概率最大化。

样本构造：Batch 内负采样（In-batch Negatives）

这是 Listwise 最关键的工程技巧：

一个训练 Batch 内的其他物品，天然构成了当前物品的负样本——无需额外采样，直接利用矩阵运算，训练效率极高。

字段 内容
用户 用户A
正样本 商品1（用户点击过）
Batch 内负样本 商品2, 3, 4, …, N（同一 Batch 内的其他物品）

损失函数：Softmax Cross-Entropy

第一步：计算正样本的概率

P(正样本)=eSposeSpos+∑j=1NeSneg,jP(\text{正样本}) = \frac{e^{S_{\text{pos}}}}{e^{S_{\text{pos}}} + \sum_{j=1}^{N} e^{S_{\text{neg},j}}} P(正样本)=eSpos​+∑j=1N​eSneg,j​eSpos​​

其中 Spos=uu⋅vi+S_{\text{pos}} = u_u \cdot v_{i+}Spos​=uu​⋅vi+​，Sneg,j=uu⋅vij−S_{\text{neg},j} = u_u \cdot v_{i_j^-}Sneg,j​=uu​⋅vij−​​。

第二步：计算负对数似然损失

Llistwise=−log⁡P(正样本)L_{\text{listwise}} = -\log P(\text{正样本}) Llistwise​=−logP(正样本)

Listwise 的关键挑战：采样偏差

Batch 内的负样本是随机采样的，不是全量物品，因此存在采样偏差。通常需要引入两个技巧：

技巧 说明
采样校正（Sampling Correction） 修正因采样导致的概率估计偏差
温度系数（Temperature） 调整 softmax 的平滑程度，控制模型对难负样本的敏感度

无论选择哪种训练方式，双塔模型在选择负样本时都很关键。

3. 正负样本的采样策略

3.1 正样本：真实行为 + 采样平衡

正样本来自用户的真实交互行为（点击/转化）。但直接使用原始行为数据会导致问题：热门商品在正样本中占比过高，模型会过度拟合热门商品，失去对中长尾商品的召回能力。

因此需要对正样本做采样平衡，核心原则是：

热门商品降采样，冷门商品重采样，让各类商品在训练中分布更均衡。 模型在各类商品上都能学到有效表征，而非只认识曝光量大的头部商品。

3.2 负样本：随机采样 + 困难样本

简单负样本：按点击热度的 0.75 次方采样

负样本的采样逻辑与正样本完全相反——热门商品更容易成为负样本：

这是 YouTube 在双塔模型论文中提出的经典方法，沿用至今。

p负(商品 i)∝(点击次数(i))0.75p_{\text{负}}(\text{商品 } i) \propto \left( \text{点击次数}(i) \right)^{0.75} p负​(商品 i)∝(点击次数(i))0.75

为什么这样设计？

  • 热门商品出现在更多用户的正样本中，因此也更有资格成为"背景负样本"
  • 让模型学会区分"用户真正感兴趣的"和"只是热度高但用户不感兴趣的"
  • 同时为后续困难负样本挖掘提供了良好的基准分布

Batch 内采样（In-batch Negatives）

与 Listwise 训练配合使用的经典技巧：

采样方式 说明
全域随机采样 从全体物品中随机抽取，基准分布
Batch 内采样 同一 Batch 内的其他物品天然构成负样本，无需额外计算

困难样本挖掘（Hard Negatives）

仅用简单负样本会让任务太简单，需引入困难样本：召回后被排序过滤掉的物品。提升模型的区分能力

⚠️ 重要禁忌：曝光未点击 ≠ 召回负样本。曝光未点击是因为被排序模型排在了后面，标签存在噪声，不能作为召回负样本使用。

3.3 正负样本采样逻辑对比

维度 正样本采样 负样本采样
采样方向 反比于点击次数（降热门、升冷门） 正比于点击次数的 0.75 次方
核心目标 让各商品分布均衡，避免头部主导 让热门商品成为负样本基准
公式 p∝1/(点击次数)αp \propto 1 / (\text{点击次数})^{\alpha}p∝1/(点击次数)α p∝(点击次数)0.75p \propto (\text{点击次数})^{0.75}p∝(点击次数)0.75
提出者 工业经验 YouTube 经典方法

4. 经典双塔模型介绍

4.1 DSSM —— 双塔语义匹配的鼻祖

论文：《Learning Deep Structured Semantic Models for Web Search using Clickthrough Data》Microsoft, CIKM 2013

DSSM（Deep Structured Semantic Model，深度结构化语义模型）是微软于 2013 年提出的深度语义匹配模型，后被广泛改造为双塔结构，成为工业界推荐系统召回阶段的主流方案。其核心思想是将用户（User）与物品（Item）映射到同一低维向量空间，通过向量相似度快速完成海量候选集的初筛。

一、核心原理与结构

DSSM 在推荐场景中采用经典的双塔架构，分为三大核心部分：

1. 输入层（Input Layer）

塔 特征类型 具体内容
用户塔 稀疏特征 用户 ID、年龄、性别、历史点击/收藏序列、设备、地域
用户塔 稠密特征 用户活跃度、历史点击率、平均停留时长
物品塔 稀疏特征 物品 ID、类目、标签、作者/商家 ID
物品塔 稠密特征 价格、评分、发布时间、热度、图片/文本向量

2. 表示层（Representation Layer / 塔身）

两侧各为独立的 DNN/MLP 网络（结构可对称或非对称，不共享权重）。

输出 说明
用户塔输出 user_emb（ddd 维向量）
物品塔输出 item_emb（ddd 维向量）

作用：将高维稀疏的原始特征，通过 Embedding + 多层全连接层，压缩映射为固定维度的稠密向量。

3. 匹配层（Matching Layer）

计算两个 Embedding 的相似度作为匹配分数：

score(u,i)=cos⁡(user_emb,item_emb)=user_emb⋅item_emb∥user_emb∥×∥item_emb∥\text{score}(u, i) = \cos(\text{user\_emb}, \text{item\_emb}) = \frac{\text{user\_emb} \cdot \text{item\_emb}}{\|\text{user\_emb}\| \times \|\text{item\_emb}\|} score(u,i)=cos(user_emb,item_emb)=∥user_emb∥×∥item_emb∥user_emb⋅item_emb​

训练目标：正样本（点击/转化）分数最大化，负样本分数最小化。

损失函数：常用交叉熵损失（Binary Cross-Entropy）或对比损失（Contrastive Loss）。

二、召回流程

DSSM 之所以成为召回标配，关键在于离线训练、线上高效检索的工程友好性：

阶段 操作 说明
离线训练 用历史行为数据（曝光→点击）训练双塔 得到 user_emb 和 item_emb
离线建库 预计算所有物品的 item_emb，存入 FAISS/Milvus 等向量数据库 构建 ANN（近似最近邻）索引
线上召回 用户请求时实时计算 user_emb 毫秒级响应
向量检索 用 user_emb 在库中快速找 Top-N 相似物品 通常召回几百～几千个
后续处理 输出候选集，送入粗排/精排

三、优缺点分析

✅ 优点 说明
性能极致 线上仅需单次用户塔推理 + ANN 检索，QPS 高、延迟低，完美适配百万/亿级候选集
解耦友好 User/Item 塔独立，可分别迭代、单独更新（如新增用户特征不影响物品塔）
通用性强 适配电商、短视频、新闻、广告等几乎所有 U2I 推荐场景
特征灵活 轻松融合 ID、行为、文本、图像等多模态特征
❌ 缺点 说明
无显式交叉 双塔独立，无法建模 User&Item 交叉特征（如"用户年龄 × 商品价格"），表达能力弱于精排模型（如 DeepFM/DIN）
信息损失 原始 DSSM 用 BOW/WordHashing，丢失序列/时序信息（已由后续变种改进）
负样本依赖 效果高度依赖负采样策略（随机/难负样本），采样不当易偏差

4.2 YouTube DNN —— 工业召回的开山之作

论文：《Deep Neural Networks for YouTube Recommendations》RecSys 2016

核心贡献

YouTube 团队首次将双塔 DNN 应用于大规模工业推荐系统，替代了此前的矩阵分解方法，并给出了大量工程实践经验。

模型架构

模型结构详解

YouTube DNN 将推荐视为多分类任务：

P(wt=i∣C)=evi⋅u∑j∈Vevj⋅uP(w_t = i | C) = \frac{e^{v_i \cdot u}}{\sum_{j \in V} e^{v_j \cdot u}} P(wt​=i∣C)=∑j∈V​evj​⋅uevi​⋅u​

其中 uuu 为用户向量，viv_ivi​ 为候选物品 iii 的向量，CCC 为用户上下文特征。

User Tower 输入特征：

特征类型 说明
观看历史 Embedding 用户看过的视频 ID → Embedding → 平均池化
搜索历史 Embedding 用户搜索词 → Embedding → 平均池化
人口属性 性别、年龄、地理位置等
Example Age 视频年龄特征（x,x2x, x^2x,x2），帮助模型平衡新老内容

训练目标：多分类 softmax，输出对所有视频的概率分布。

Serving 阶段：

  • 物品向量 viv_ivi​ 离线计算好，构建 ANN 索引
  • 用户请求时实时计算用户向量 uuu
  • 用 uuu 在 ANN 索引中检索 Top-K 候选物品

YouTube DNN 的关键创新

创新点 说明
多分类 softmax 将召回建模为大规模多分类问题（百万~亿级类别）
Example Age 视频上传时间作为特征，缓解推荐系统对新内容的天然偏向
离线/在线分离 物品向量离线算好，在线只算用户向量，兼顾效率与实时性
历史序列建模 首次将用户观看/搜索历史序列融合进召回模型

YouTube DNN 的局限性

  • 用户向量由平均池化得到，无法建模兴趣的时序关系
  • 用户只有一个向量，无法表达多兴趣

4.3 MIND —— 多兴趣建模的开创者

论文：《Multi-Interest Network with Dynamic Routing for Recommendation at Alibaba》CIKM 2019

核心贡献

MIND（Multi-Interest Network with Dynamic Routing）由阿里巴巴提出，首次将 Capsule Network（胶囊网络） 和 Dynamic Routing（动态路由） 引入推荐召回，解决了单用户向量无法表达多兴趣的问题。

模型架构

模型结构详解

核心洞察：一个用户可能有多种兴趣（买衣服、买电器、看新闻），用一个向量无法充分表达。

Multi-Interest Extractor Layer（多兴趣提取层）：

MIND 将 Capsule Network 的动态路由机制应用于推荐：从用户行为序列中提取 KKK 个不同的兴趣向量。

动态路由的迭代公式：

mjk=Wkj⋅ei（将物品向量映射到 Capsule）sjk=exp⁡(bjk)∑texp⁡(bjt)（计算路由权重）rk=∑jsjk⋅mjk（加权求和）vk=squash(rk)（Squash 激活，保证向量长度）\begin{aligned} m_{jk} &= W_{kj} \cdot e_i \quad \text{（将物品向量映射到 Capsule）} \\ s_{jk} &= \frac{\exp(b_{jk})}{\sum_t \exp(b_{jt})} \quad \text{（计算路由权重）} \\ r_k &= \sum_j s_{jk} \cdot m_{jk} \quad \text{（加权求和）} \\ v_k &= \text{squash}(r_k) \quad \text{（Squash 激活，保证向量长度）} \end{aligned} mjk​sjk​rk​vk​​=Wkj​⋅ei​（将物品向量映射到 Capsule）=∑t​exp(bjt​)exp(bjk​)​（计算路由权重）=j∑​sjk​⋅mjk​（加权求和）=squash(rk​)（Squash 激活，保证向量长度）​

其中 squash(r)=∥r∥21+∥r∥2⋅r∥r∥\text{squash}(r) = \frac{\|r\|^2}{1 + \|r\|^2} \cdot \frac{r}{\|r\|}squash(r)=1+∥r∥2∥r∥2​⋅∥r∥r​，使短向量压缩至接近 0，长向量接近 1。

Label-aware Attention（标签感知注意力）：训练时，用目标物品作为 Query，对 KKK 个兴趣向量做 Attention，选择最相关的兴趣参与损失计算：

Serving 阶段：

  • 直接用所有 KKK 个兴趣向量并行做 ANN 检索
  • 每路返回 Top-N，结果合并去重
  • 多路召回可以覆盖用户多种兴趣，推荐结果更丰富

5. 双塔模型的线上应用

5.1 为什么 User 塔实时推理，Item 塔离线推理？

推理对象 推理方式 原因
User 塔 ✅ 实时推理 用户量级有限（百万~亿级），每次请求只需算一次
Item 塔 ✅ 离线推理 物品数量庞大（亿级），每次请求都算一遍成本极高

核心逻辑：一次用户请求只需计算一次用户向量，但系统中有数亿个物品。若每个物品实时过一遍 DNN，计算量是灾难性的。

因此：物品向量提前算好存起来，用户请求时实时算用户向量，做向量检索。

5.2 新物品的索引更新

双塔模型利用物品内容特征，新品上线无需重新训练模型，只需：

  1. 提取新物品的内容特征（类目、价格、标签等）
  2. 用训练好的 Item 塔 DNN 做一次前向传播
  3. 产出新物品向量，写入 ANN 索引

这大幅缓解了新品冷启动问题，是双塔模型相比矩阵分解的重要优势。

YouTube DNN/MIND：新物品的 Embedding 查询

YouTube DNN/MIND 没有独立的 Item 塔，item embedding 就是 softmax 权重，无法直接推理新物品，必须重新训练或用旁路方案。比如：

  • 方案1：给新物品 “随机初始化 embedding”（简单粗暴），新物品直接用随机正态分布向量，等下次训练再更新
  • 方案2：搭一个旁路 Item 塔（业界最常用），主模型仍然用 YouTubeDNN / MIND 训练（效果最好）。额外训练一个小的 Item 塔，输入：类目、标签、作者、文本、图片向量等。新物品来了 → 过 Item 塔 → 得到临时 embedding，等该物品有行为后，下一轮训练自动进入 softmax 权重。

5.3 完整召回流程

阶段 操作 说明
离线 Item 塔前向传播 遍历所有物品，产出向量，存入 ANN 索引库
在线 User 塔实时推理 用户请求 → 提取实时特征 → User DNN → 用户向量
在线 ANN 向量检索 用户向量 → ANN 索引 → Top-K 个最相似物品
后续 粗排/精排 召回的 K 个物品进入排序阶段

6. 总结

双塔模型的核心贡献：

贡献 说明
内容特征融合 将 ID + 内容特征同时建模，泛化能力大幅提升
深度表征学习 用 DNN 学非线性用户/物品表征
工程最优范式 离线物品向量 + 在线用户向量 + ANN 高效检索

一句话总结：双塔模型用两个 DNN 分别学习用户和物品的丰富表征，训练时用 Pairwise 对抗学习，在线服务时物品离线推理、用户实时推理，通过 ANN 向量检索高效完成召回。

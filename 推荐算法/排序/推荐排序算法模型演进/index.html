推荐排序算法模型演进

发表于2026-04-05|更新于2026-04-06|推荐算法排序

排序是推荐系统的核心环节。本文系统梳理从 LR、GBDT 到 Wide&Deep、DIN 等排序模型的完整演进脉络，覆盖特征体系、特征处理、传统排序模型、深度学习排序模型、基础的行为序列建模等内容。

1. 排序在推荐漏斗中的位置与核心使命

一个完整的推荐系统通常由召回（Retrieval）→ 粗排（Pre-Rank）→ 精排（Ranking）→ 重排（Re-rank） 四层构成，形状像一个倒漏斗：

  • 召回：从海量物品库（百万至亿级）中快速筛选出候选集（千至万级），注重吞吐量和相关性
  • 粗排：对召回候选集做初步排序，输出约 千级 规模
  • 精排：对候选物品逐一打分，输出约 千级 规模，进入重排阶段
  • 重排：业务规则干预、多样性控制，最终输出 几十个 精品结果展示给用户

在整条漏斗中，精排是特征使用最丰富、模型精度要求最高的环节，任务核心是对单个物品的打分预测。如漏斗右侧所示，精排阶段的核心处理链路为：用户特征、物品特征、交叉特征、上下文特征经 Embedding 层后进入多塔 DNN，最终输出 CTR/CVR 等预估分数用于排序。

2. 排序模型常用特征

排序模型相比协同过滤最大的优势在于能够融合多维异构特征，包括用户画像、用户行为、物品画像、用户x物品交叉、上下文信息等，这也是排序效果远优于协同过滤的根本原因。

用户侧特征

画像特征：描述用户稳定属性的特征，更新频率较低。

  • 示例：年龄、性别、职业、设备类型、会员等级、兴趣标签体系

行为序列特征：用户在平台上的历史行为，是排序模型中最有区分度的特征。

  • 示例：点击物品 ID 序列、购买物品 ID 序列、浏览品类序列、搜索词序列

统计特征：基于用户历史行为聚合得到的数值指标。

  • 示例：用户历史点击率、用户平均停留时长、用户活跃天数、用户消费能力分层

物品侧特征

画像特征：物品的固有属性，通常稳定或更新频率极低。

  • 示例：物品 ID、品类 ID、品牌 ID、价格区间、上架时间

统计特征：基于历史数据统计得到的物品层面指标。

  • 示例：物品历史 CTR、物品历史 CVR、物品曝光量、物品销量

交叉特征（用户 × 物品）

描述用户和物品之间特定交互关系的特征，是增强模型表达能力的关键。

  • 示例：用户年龄段 × 物品品类、用户兴趣标签 × 物品标签

上下文特征

描述推荐发生时环境上下文的特征，是模型实现场景化适配的关键。

  • 示例：请求时间（工作日/节假日）、推荐位排序、入口来源、设备类型

3. 特征处理方式

前面的特征，按类型可以再分为离散型和连续型特征，对不同类型的特征有不同的处理方式。

3.1 离散型特征

离散特征（Categorical Feature）取值来自有限集合，直接 One-Hot 编码会导致维度爆炸（物品 ID 数量可达百万级），通常用embedding来表示。

处理方式——Embedding：将离散特征映射到低维稠密向量空间：

Embedding(xi)∈Rd\text{Embedding}(x_i) \in \mathbb{R}^d Embedding(xi​)∈Rd

典型示例：

  • 性别（3维 One-Hot → 2维 Embedding）
  • 物品 ID（百万维 One-Hot → 32维 Embedding）

3.2 连续型特征

连续特征（Continuous Feature）取值为实数，通常跨度大且分布不规则。

处理方式一：直接归一化输入

  • Min-Max 缩放：x′=(x−xmin⁡)/(xmax⁡−xmin⁡)x' = (x - x_{\min}) / (x_{\max} - x_{\min})x′=(x−xmin​)/(xmax​−xmin​)
  • Z-Score 标准化：x′=(x−μ)/σx' = (x - \mu) / \sigmax′=(x−μ)/σ

处理方式二：分桶（Bucketization）离散化

  • 将连续值划分子区间映射为离散桶 ID，再做 Embedding
  • 划分方式可基于业务经验（价格区间）或数据驱动（等频/等宽分桶）

4. 传统排序模型演进

在深度学习浪潮之前，以逻辑回归（LR） 为代表的统计学习模型主导了推荐排序领域。相比协同过滤仅利用用户-物品交互矩阵，这些模型能够融合多维特征实现更精准的预估。

4.1 LR——简单而坚挺的基石

逻辑回归（Logistic Regression，LR）将各特征加权求和，通过 Sigmoid 函数映射为点击概率：

P(click)=σ(w0+∑i=1nwixi)P(\text{click}) = \sigma\left(w_0 + \sum_{i=1}^{n} w_i x_i\right) P(click)=σ(w0​+i=1∑n​wi​xi​)

优点：简单、直观、易解释、易并行、收敛快，在特征工程充分时效果稳定。
缺点：表达能力有限，无法自动捕捉特征交叉——"用户喜欢运动品牌 × 物品是运动鞋"这种交叉信号，LR 无法自动学到。

4.2 POLY2——显式特征交叉的初次尝试

POLY2 对所有特征做两两交叉，并赋予独立权重：

y^=w0+∑i=1nwixi+∑i=1n∑j=i+1nwij⋅xixj\hat{y} = w_0 + \sum_{i=1}^{n} w_i x_i + \sum_{i=1}^{n} \sum_{j=i+1}^{n} w_{ij} \cdot x_i x_j y^​=w0​+i=1∑n​wi​xi​+i=1∑n​j=i+1∑n​wij​⋅xi​xj​

问题：权重参数从 nnn 急剧上升到 n2n^2n2 级别。对于高基数离散特征（如物品 ID 可达百万级），训练复杂度完全不可接受，交叉特征的稀疏问题也极为严重。

4.3 FM——隐向量分解的破局者

2010 年，Rendle 提出因子分解机（Factorization Machine，FM），用特征隐向量内积替代了 POLY2 中显式的交叉权重：

y^=w0+∑i=1nwixi+∑i=1n∑j=i+1n⟨vi,vj⟩⋅xixj\hat{y} = w_0 + \sum_{i=1}^{n} w_i x_i + \sum_{i=1}^{n} \sum_{j=i+1}^{n} \langle \mathbf{v}_i, \mathbf{v}_j \rangle \cdot x_i x_j y^​=w0​+i=1∑n​wi​xi​+i=1∑n​j=i+1∑n​⟨vi​,vj​⟩⋅xi​xj​

引入隐向量 vi∈Rk\mathbf{v}_i \in \mathbb{R}^kvi​∈Rk（k≪nk \ll nk≪n），将 n2n^2n2 个权重参数压缩为 n×kn \times kn×k 个。类比矩阵分解的思想，两个特征的交叉权重不再独立学出，而是由各自隐向量的内积近似表示——即便某个交叉特征从未在训练数据中出现，FM 仍可通过隐向量内积给出合理预测。

4.4 GBDT——树模型的特征挖掘能力

梯度提升决策树（GBDT） 族模型（XGBoost、LightGBM）天然具备自动特征选择和组合能力。GBDT 通过多轮建树，每棵树都在拟合上一轮的残差，最终通过所有树的叶子节点路径编码隐式表达特征组合。

4.5 GBDT + LR——Facebook 的范式转变

2014 年，Facebook 在论文"Practical Lessons from Predicting Clicks on Ads at Facebook"中提出GBDT + LR 架构，是传统排序模型时代最具影响力的工作之一：

核心思想：用 GBDT 进行特征筛选和非线性组合，将 GBDT 的叶子节点路径编码为高维离散向量，作为 LR 的输入。

GBDT + LR 的意义在于：将树模型的非线性特征组合能力与 LR 的可解释性相结合，大幅降低了对人工特征工程的依赖，深刻影响了后续深度学习时代的特征交叉建模思路。

5. 深度学习排序模型演进

2012 年 AlexNet 引爆深度学习浪潮，2016 年起搜推广正式进入深度学习时代。深度模型的核心突破在于：端到端地完成从稀疏特征到稠密向量再到高阶特征交叉的全过程，大幅降低了对人工特征工程的依赖。

深度排序模型在推荐系统中呈现出以下特点：

在特征处理上，能够高效处理高维稀疏特征，自动学习低阶与高阶特征交叉，同时对用户行为序列等复杂结构化特征友好。

在模型结构上，网络设计日趋强大与复杂，表达能力显著提升；结构灵活可插拔，可根据业务目标自由组合模块、适配不同场景。

在技术演进上，深度排序模型不断吸收NLP、计算机视觉领域的前沿进展（如注意力机制、Transformer、预训练范式等），实现快速迭代与持续优化。

整体演进方向可概括为四条主线：

  1. 特征交叉与融合（DeepCrossing、Wide&Deep、DeepFM等）
  2. 用户行为序列建模（DIN、DIEN、Transformer序列模型）
  3. 多目标优化（SharedBottom、MMoE、PLE、ESMM）
  4. 大模型与预训练、生成式推荐

本章节主要介绍，特征交叉与融合相关的工作，是深度学习和推荐的初步结合。后续文章绘更多的介绍行为序列建模、排序多目标优化和LLM4Rec等方向的工作。

5.1 AutoRec——自编码器与协同过滤的碰撞

AutoRec（2015）是将自编码器（AutoEncoder）用于协同过滤的轻量深度推荐模型，核心是用单隐层自编码器重构用户/物品的评分向量，预测缺失评分。

  • 核心思想：输入用户/物品的评分向量（缺失值填0），经编码器压缩为低维隐向量，再经解码器重构完整评分矩阵，用重构结果补全未评分项。
  • 两种形式：
    • I-AutoRec（物品侧）：输入物品 iii 的评分向量 r(i)\mathbf{r}^{(i)}r(i)，编码后得到隐藏层表征 h=f(W⋅r(i)+b)\mathbf{h} = f(\mathbf{W} \cdot \mathbf{r}^{(i)} + \mathbf{b})h=f(W⋅r(i)+b)，解码输出所有用户对物品 iii 的评分预测
    • U-AutoRec（用户侧）：以用户为编码单元的对应版本
  • 结构：输入层 → 隐层（编码）→ 输出层（解码），仅单隐层、激活函数常用Sigmoid。
  • 公式：r^=f(W⋅g(Vr+μ)+b)\hat{r}=f(W\cdot g(Vr+\mu)+b)r^=f(W⋅g(Vr+μ)+b)

AutoRec 是深度学习与推荐系统结合的开创性探索，但精度表现一般，其意义在于打开了深度学习改造推荐系统的大门。

5.2 Deep Crossing——端到端深度排序奠基之作

2016 年，微软发表论文"Deep Crossing: Web-Scale Modeling without Manual Feature Crafting"，系统性解决了三大问题：

  1. 稀疏特征稠密化：通过特征 Embedding 将高维稀疏离散特征映射为低维稠密向量
  2. 特征自动交叉：不再依赖人工特征工程，通过多层残差网络对特征向量各维度进行高阶交叉组合
  3. 残差结构：采用多层残差结构（Stacking 层 + Multiple Residual Units），用残差连接解决深度网络梯度消失，让模型能学高阶非线性特征交互。

Deep Crossing 是深度排序模型的奠基之作，其框架沿用至今。

5.3 Wide&Deep——记忆与泛化的权衡艺术

2016 年，Google 在论文"Wide & Deep Learning for Recommender Systems"中提出的 Wide&Deep，是深度排序模型领域影响力最大、商业化最成功的架构。

  • Wide 部分（左侧）

    • 输入：原始特征 + 人工交叉特征（如用户A×物品B）
    • 结构：单层广义线性模型
    • 作用：记忆历史共现规则（强规则、高频组合）

  • Deep 部分（右侧）

    • Embedding层：稀疏类别特征 → 低维稠密向量
    • MLP层：多层全连接（ReLU），自动学习高阶隐式特征交叉
    • 作用：泛化，适配新特征、冷启与长尾样本

  • 输出

    • Wide输出 + Deep输出 → Sigmoid → CTR概率
    • 损失：对数交叉熵（LogLoss）

Wide&Deep 深刻揭示了推荐系统的核心矛盾：精确记忆 vs. 泛化推理，并通过双通道架构优雅地解决了这一矛盾。

5.4 Deep&Cross Network（DCN）——Cross 网络替代 Wide

Deep&Cross Network（DCN）由 Google 团队于 2017 年提出（DCN v2 为后续改进版本），用Cross Network 替代了 Wide 侧的手动特征工程。Cross 网络通过多层交叉显式学习高阶特征交互：

xl+1=x0⊙(Wl⊤xl)+bl+xl\mathbf{x}_{l+1} = \mathbf{x}_0 \odot \left(\mathbf{W}_l^\top \mathbf{x}_l\right) + \mathbf{b}_l + \mathbf{x}_l xl+1​=x0​⊙(Wl⊤​xl​)+bl​+xl​

其中 ⊙\odot⊙ 表示逐元素 Hadamard 乘积，Wl∈Rd×d\mathbf{W}_l \in \mathbb{R}^{d \times d}Wl​∈Rd×d 为第 lll 层的可学习权重矩阵。

每一层中，输入 xl\mathbf{x}_lxl​ 与原始输入 x0\mathbf{x}_0x0​ 做交叉，使高阶组合信息逐层累积。Cross 网络参数量远小于同等深度的全连接网络，但表达能力更强——是对"宽模型"思路的优雅升级。

5.5 DeepFM——FM 与深度学习的端到端整合

DeepFM 将 FM 模型与深度神经网络整合，用FM 组件替代 Wide&Deep 中的 Wide 侧：

  • FM 侧：通过二阶特征隐向量内积，显式建模所有特征的两两交叉
  • Deep 侧：通过 DNN 建模高阶隐式交叉

DeepFM 无需像 Wide&Deep 那样人工设计 Wide 侧的特征组合，实现了真正端到端的特征自动交叉，同时兼顾低阶（FM 一阶、二阶）和高阶（DNN 多隐层）交叉。

6. 用户行为序列建模

用户行为序列（User Behavior Sequence）是推荐系统中信息量最大、区分度最高的特征之一。用户的历史点击、购买、浏览等行为蕴含着强烈的兴趣信号，几乎是精排模型的必用特征。本章仅介绍最简单的行为序列建模方式。

6.1 YouTube DNN——Pooling 聚合方案

2016 年，YouTube 在论文"Deep Neural Networks for YouTube Recommendations"中首次将 DNN 应用于视频推荐。对于用户行为序列的处理，将历史观看视频的 Embedding 向量做平均 Pooling：

ewatch=1N∑i=1Neiwatch\mathbf{e}_{\text{watch}} = \frac{1}{N} \sum_{i=1}^{N} \mathbf{e}_i^{\text{watch}} ewatch​=N1​i=1∑N​eiwatch​

平均 Pooling 简单高效，但无法区分不同行为对当前候选物品的不同贡献度——一篇游戏视频的历史观看，对推荐游戏类视频和推荐美食类视频的贡献度显然不同，但 Pooling 将它们等同对待。

6.2 DIN——注意力机制加权聚合

2018 年，阿里巴巴团队提出Deep Interest Network（DIN），引入注意力机制对行为序列进行加权聚合：

eseq=∑i=1Na(eitemi,etarget)⋅eitemi\mathbf{e}_{\text{seq}} = \sum_{i=1}^{N} a\left(\mathbf{e}_{\text{item}_i}, \mathbf{e}_{\text{target}}\right) \cdot \mathbf{e}_{\text{item}_i} eseq​=i=1∑N​a(eitemi​​,etarget​)⋅eitemi​​

DIN 的注意力是用一个小神经网络 MLP 来拟合权重，输入是历史行为 embedding 和 候选商品 embedding 及其交叉。

把 4 类信息拼在一起输入 MLP：

  1. 历史商品向量：ei\boldsymbol{e}_iei​
  2. 候选商品向量：va\boldsymbol{v}_ava​
  3. 外积/元素乘：ei⊙va\boldsymbol{e}_i \odot \boldsymbol{v}_aei​⊙va​
  4. 差值：∣ei−va∣|\boldsymbol{e}_i - \boldsymbol{v}_a|∣ei​−va​∣

拼接后：

xi=[ei,va,ei⊙va,∣ei−va∣]\boldsymbol{x}_i = [\boldsymbol{e}_i, \boldsymbol{v}_a, \boldsymbol{e}_i\odot\boldsymbol{v}_a, |\boldsymbol{e}_i-\boldsymbol{v}_a|] xi​=[ei​,va​,ei​⊙va​,∣ei​−va​∣]

通过一个单层（或小多层）MLP + 激活，输出标量分数：

ai=MLP(xi)a_i = \text{MLP}(\boldsymbol{x}_i) ai​=MLP(xi​)

再经过 softmax 归一化，得到最终权重：

wi=exp⁡(ai)∑jexp⁡(aj)w_i = \frac{\exp(a_i)}{\sum_j \exp(a_j)} wi​=∑j​exp(aj​)exp(ai​)​

加权得到用户兴趣向量

u=∑i=1Nwi⋅ei\boldsymbol{u} = \sum_{i=1}^N w_i \cdot \boldsymbol{e}_i u=i=1∑N​wi​⋅ei​

即DIN 注意力分数 = 用 MLP 拟合历史商品与候选商品的相关性，输入包含向量本身、差、元素积，再 softmax 归一化。

DIN 的核心创新在于：引入序列兴趣与目标物品的匹配度作为权重，让模型学会区分不同行为对当前推荐目标的不同价值——与候选物品更相关的历史行为将获得更高的注意力权重。

注：DIEN 的序列兴趣演化建模、Capsule 网络的多兴趣表征（MIND）、基于 Transformer 的序列建模（SIM、STD）等更复杂的方案，我们将在后续博文中展开详细介绍。

7. 结语

从 LR 到 GBDT+LR，从 Wide&Deep 到 DIN，排序模型的演进始终围绕一个核心命题：如何在有限的计算资源下，更精准地建模用户-物品之间的匹配关系。

传统机器学习时代，核心瓶颈是特征交叉的稀疏性问题——FM 通过隐向量分解优雅地解决了这一问题。深度学习时代的核心突破，则是端到端的特征表示学习能力，让模型从繁重的人工特征工程中解放出来。

而用户行为序列建模，是整个演进链条中最具生命力的方向之一。从简单的平均 Pooling，到注意力加权的 DIN，再到更复杂的序列建模方案——如 DIEN 的序列兴趣演化机制、MIND 的多兴趣 Capsule 表征、以及基于 Transformer 的序列建模（SIM、STD 等）——行为序列建模始终是推荐排序模型最活跃的研究方向，也将是这一领域持续演进的主线之一。

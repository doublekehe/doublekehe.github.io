推荐多目标排序建模

发表于2026-04-04|更新于2026-04-04|推荐算法排序

推荐系统的真实业务从来不是单一目标——如何让 CTR、CVR、GMV 等多个目标协同优化，是推荐系统的核心难题之一。

1. 多目标排序的背景

1.1 业务目标天然多元

真实的推荐业务场景中，用户行为是多维的，衡量指标也是多元的：

场景 典型目标 说明
音乐推荐 完播率、收藏率、分享率 用户是否真的在消费内容
电商推荐 点击率（CTR）、转化率（CVR）、GMV 曝光→点击→成交的完整链路
视频推荐 播放完成率、互动率、留存率 内容质量和用户参与度并重

1.2 目标之间的冲突

多目标之间并非总是协同的，它们往往存在天然冲突：

  • CTR ↑ vs CVR ↓：低门槛内容（八卦、擦边）CTR 高但 CVR 低；高门槛内容（专业教程）CTR 低但 CVR 高
  • 短期点击 vs 长期留存：单纯优化 CTR 会导致标题党泛滥，用户新鲜感消退后留存下降
  • GMV 最大化 vs 用户体验：过度推销高毛利商品会损害用户体验

因此，多目标排序的本质不是"选最优"，而是在多个目标之间找到帕累托最优的平衡点。

1.3 传统单目标建模的困境

传统做法是为每个目标独立训练一个模型：

  • 成本高：N 个目标需要维护 N 个模型，特征工程、训练、部署成本线性增长
  • 知识孤岛：各模型独立优化，无法共享底层特征表示，造成信息浪费
  • 目标对齐困难：多模型分数融合时，量纲不一致，调参依赖人工经验

多目标排序建模正是为了解决这些问题——让多个目标在同一模型中联合学习，实现参数共享、知识迁移和多目标协同优化。

2. 多目标排序的建模方式

多目标建模有四种范式，核心区别在于"共享程度"和"任务之间的关系"如何利用。

2.1 完全独立建模

每个任务单独训练一个模型，完全不共享参数。参数量大，但任务之间不会相互干扰。工业界通常只用作 baseline。

2.2 硬共享参数（SharedBottom）

所有任务共享同一个底层 DNN，各自通过轻量级 Tower 输出。参数少、训练快，但所有任务的梯度在共享层汇合，任务差异大时会出现跷跷板现象。

2.3 软共享 / 部分共享

只共享底层 Embedding，上层网络实现部分或自适应共享，是工业界主流方案：

  • Multi-Tower：只共享 Embedding，塔的每一层完全独立，适合任务差异较大的场景
  • MMoE：多个 Expert + 每个任务独立的门控网络，自适应选择 Expert 组合，是软共享的代表
  • PLE：显式分离共享专家和任务专属专家，避免好任务被差任务拖垮，是 PLE 的核心优势

2.4 结构约束型（ESMM）

任务之间不是简单共享参数，而是有业务上的数学约束。最典型的例子是 ESMM：转化 = 点击 × 点击后转化，用 CTCVR = pCTR × pCVR 的乘法关系在全链路监督 CTR 和 CVR，既解决了数据稀疏问题，也避免了样本选择偏差。

2.5 四种建模方式对比

方式 共享程度 优点 缺点 代表
完全独立 无共享 不冲突 参数量大、无知识共享 N 个单独模型
硬共享（SharedBottom） 底层完全共享 快、省参 任务冲突严重 SharedBottom
软共享（MMoE / PLE） 部分/自适应共享 效果强、稳定 结构复杂 MMoE、PLE
结构约束型 共享+业务约束 解决样本偏差 仅适用链式任务 ESMM

3. 多目标排序部分算法介绍

3.1 Multi-Tower（多塔模型）

多塔模型是软共享范式中最简洁、最直接的一种，与 SharedBottom 的本质区别在于：共享的只有 Embedding 层，塔本身的每一层参数完全独立不共享。

设共享 Embedding 输出为 e=Embedding(x)e = \text{Embedding}(x)e=Embedding(x)，任务 kkk 的塔输出为：

hk=ftowerk(e),y^k=σ ⁣(Wk⋅hk+bk)h^k = f_{\text{tower}}^k(e), \quad \hat{y}_k = \sigma\!\left(W^k \cdot h^k + b^k\right) hk=ftowerk​(e),y^​k​=σ(Wk⋅hk+bk)

其中 ftowerkf_{\text{tower}}^kftowerk​ 是任务 kkk 塔的完整 DNN 参数，各任务的塔之间互不干扰、独立学习。

总 loss 为各任务 loss 的加权求和：

Ltotal=∑k=1Kwk⋅Lk(y^k,yk)L_{\text{total}} = \sum_{k=1}^{K} w_k \cdot L_k(\hat{y}_k, y_k) Ltotal​=k=1∑K​wk​⋅Lk​(y^​k​,yk​)

与 SharedBottom 的核心区别：

对比维度 SharedBottom Multi-Tower
共享位置 中间激活层（MLP） 仅输入 Embedding
塔参数 轻量级（1~2层） 完全独立，可很深
梯度冲突 严重（所有任务共享层打架） 无（塔之间完全隔离）
表达能力 受限于共享层容量 各任务独立充分表达

适用场景： 任务之间差异较大、对各自目标的学习深度要求不同，但共享 Embedding 特征表示仍有收益的场景。

3.2 MMoE（Multi-Gate Mixture of Experts）

论文：Ma et al., “Modeling Task Relationships in Multi-task Learning with Multi-gate Mixture-of-Experts”, KDD 2018.

如图中©所示，MMoE 在结构上做了两步升级：

  • (b) 单门控 MoE：多个 Expert 共享，但所有任务共用同一个 Softmax 权重，任务之间无法区分
  • © MMoE：每个任务配备独立 Gate，自适应决定从哪些 Expert 汲取知识，实现真正的软参数共享

设共有 NNN 个 Expert，任务 kkk 的门控输出为加权组合：

gk(x)=Softmax ⁣(Wgatek⋅x)∈RNg^k(x) = \text{Softmax}\!\left(W_{\text{gate}}^k \cdot x\right) \in \mathbb{R}^N gk(x)=Softmax(Wgatek​⋅x)∈RN

fk(x)=∑i=1Ngik(x)⋅fi(x)f^k(x) = \sum_{i=1}^{N} g_i^k(x) \cdot f_i(x) fk(x)=i=1∑N​gik​(x)⋅fi​(x)

最终输出：

y^k=σ ⁣(Wk⋅fk(x)+bk)\hat{y}_k = \sigma\!\left(W^k \cdot f^k(x) + b^k\right) y^​k​=σ(Wk⋅fk(x)+bk)

极化现象（Polarization）： 当任务相关性低时，门控容易收敛到某个 Expert 权重接近 1、其他接近 0，MMoE 退化为 SharedBottom。解决：对 Softmax 门控施加 Dropout（10%~20%）。

3.3 PLE（Progressive Layered Extraction）

论文：Tang et al., “Progressive Layered Extraction (PLE): A Novel Multi-Task Learning for CTR Prediction”, RecSys 2020.

3.3.1 CGC（Customized Gate Control，基础版）

核心：把专家分成两类，从结构上强制分离

MMoE 的问题是所有任务共享所有 Expert，当任务之间差异大时，共享 Expert 容易被某个任务"带偏"，拖累其他任务。CGC 的解决思路是把 Expert 物理拆开：

  • Shared Experts（蓝色）：所有任务共用，学通用特征
  • Task-Specific Experts（粉色/绿色）：每个任务独立，只受本任务梯度更新

例：CTR 任务 → Experts_CTR + Experts_Shared；CVR 任务 → Experts_CVR + Experts_Shared

门控规则（关键）：

每个任务门控只看自己专属 + 共享专家，完全不看其他任务的专属专家——从结构上杜绝跨任务干扰。

CGC 公式（任务 k）：

fk=gk ⁣([Ek;Es])⋅[Ek;Es]f_k = g_k\!\left(\left[E_k ; E_s\right]\right) \cdot \left[E_k ; E_s\right] fk​=gk​([Ek​;Es​])⋅[Ek​;Es​]

其中：EkE_kEk​ 为任务 kkk 的专属专家输出，EsE_sEs​ 为共享专家输出，gkg_kgk​ 为任务 kkk 的门控（只对 EkE_kEk​ 和 EsE_sEs​ 加权），⊙\odot⊙ 为逐元素加权求和。

3.3.2 PLE（CGC 多层堆叠，完整版）

CGC 是单层结构，表达能力有限。PLE 将多层 CGC 模块纵向堆叠，通过逐层渐进分离，实现更强的特征提取能力：

图注： (a) Hard Sharing → (e) CGC（PLE 单层版本） → (h) ML-MMoE → (i) PLE（多层堆叠，正式版本）

每层都包含：专属 Expert + 共享 Expert + 专属 Gate + 共享 Gate，共享门控把所有 Expert 的输出融合后传给下一层，实现跨层信息交互，同时保护任务专属知识不被其他任务污染。

3.4 ESMM（Entire Space Multi-Task Model）

论文：Ma et al., “Entire Space Multi-Task Modeling: An Effective Approach for Conversion Rate Prediction”, SIGIR 2018.

一句话核心：不直接学 CVR，而是用 CTR + CTCVR 两个全空间任务间接算出 CVR，让 CVR 也能在全曝光样本上学习。

传统 CVR 模型的两大痛点

痛点 说明
样本选择偏差（SSB） 传统 CVR 只在点击样本上训练，但推理在全曝光样本上——训练空间 ≠ 推理空间，预测必然有偏
数据稀疏（DS） 曝光 ≫ 点击 ≫ 转化，CVR 正样本（点击+转化）极为稀少，模型难以收敛

核心概率原理

用户行为链：曝光 → 点击 → 转化

p(click∧convert)=p(click)×p(convert∣click)p(\text{click} \land \text{convert}) = p(\text{click}) \times p(\text{convert} \mid \text{click}) p(click∧convert)=p(click)×p(convert∣click)

即：

pCTCVR=pCTR×pCVR⟹pCVR=pCTCVRpCTRp_{\text{CTCVR}} = p_{\text{CTR}} \times p_{\text{CVR}} \quad \Longrightarrow \quad p_{\text{CVR}} = \frac{p_{\text{CTCVR}}}{p_{\text{CTR}}} pCTCVR​=pCTR​×pCVR​⟹pCVR​=pCTR​pCTCVR​​

ESMM 的创新点：不直接训练 CVR（没有监督信号），而是同时训练 CTR 和 CTCVR 两个全空间任务，CVR 作为中间变量由两者相除间接得到——从根本上消除样本选择偏差，并利用海量曝光样本缓解稀疏。

模型结构：双塔共享 Embedding

  • CTR 塔（右）：在全曝光样本上训练，输出 pCTRp_{\text{CTR}}pCTR​
  • CVR 塔（左）：无直接 Loss，用 CTR 塔共享的 Embedding，输出 pCVRp_{\text{CVR}}pCVR​
  • CTCVR：直接监督信号，pCTCVR=pCTR×pCVRp_{\text{CTCVR}} = p_{\text{CTR}} \times p_{\text{CVR}}pCTCVR​=pCTR​×pCVR​（逐元素相乘）

共享 Embedding 的意义：CVR 塔复用 CTR 塔学到的 Embedding，让 CVR 也能从曝光未点击样本中学习，彻底解决数据稀疏。

三大优势

优势 说明
✅ 彻底解决 SSB 训练和推理都在全曝光空间，不再用有偏的点击子集
✅ 根治数据稀疏 CTR 海量监督信号学到的 Embedding，被 CVR 间接复用
✅ 概率一致 严格遵循"曝光→点击→转化"的因果顺序，可解释性强

3.5 算法对比总结

算法 共享方式 任务解耦能力 数据稀疏处理 适用场景
Multi-Tower 仅共享 Embedding，塔独立 ✅ 强（塔间完全隔离） ❌ 无 任务差异大、各自需深度学习
MMoE 软共享（门控加权） ✅ 中（极化风险） ❌ 无 任务有一定相关性
PLE 显式分离共享/专属专家 ✅✅ 强 ❌ 无 任务差异大、跷跷板现象
ESMM 共享 Embedding + 全链路建模 ✅ 中 ✅✅ 强（CTCVR 辅助监督） CVR 稀疏、链式转化场景
不确定性加权 叠加于上述各框架 取决于底层框架 ✅ 自动调整 任务难度差异大

4. 多目标分数融合

多目标建模完成后，各任务会输出独立的预测分数，如何将这些分数融合为一个综合分数，是工程落地的关键。

4.1 加权融分

最通用的方式是各目标分数加权求和：

sfinal=∑k=1Kwk⋅y^k,∑wk=1,wk≥0s_{\text{final}} = \sum_{k=1}^{K} w_k \cdot \hat{y}_k, \quad \sum w_k = 1, \quad w_k \ge 0 sfinal​=k=1∑K​wk​⋅y^​k​,∑wk​=1,wk​≥0

权重 wkw_kwk​ 的设置方式：

方法 说明 优缺点
人工设定 业务专家根据 KPI 直接指定 简单直接，但无法适应数据分布变化
Grid Search 在验证集上网格搜索最优组合 可靠，但当目标数 > 3 时搜索空间急剧膨胀
进化算法 用遗传算法/贝叶斯优化搜索权重 可处理高维，但调参成本高

4.2 乘积融分

各目标分数连乘，综合考虑多个目标的均衡表现：

sfinal=∏k=1Ky^kwks_{\text{final}} = \prod_{k=1}^{K} \hat{y}_k^{w_k} sfinal​=k=1∏K​y^​kwk​​

乘积融分的优势在于：任一目标的分数趋近于 0，最终分数都会大幅下降，有效防止"某个目标极差但其他目标极高"的失衡情况。相比加权求和，乘积融分对短板的惩罚更重。

4.3 学习融合权重

不依赖人工设定，而是用一个轻量级模型（如 Logistic Regression 或单层 MLP）来学习最优融合：

sfinal=MLP ⁣([y^1,y^2,…,y^K])s_{\text{final}} = \text{MLP}\!\left([\hat{y}_1, \hat{y}_2, \ldots, \hat{y}_K]\right) sfinal​=MLP([y^​1​,y^​2​,…,y^​K​])

或者将各目标分数拼接后输入排序模型（Learning to Rank）进行统一排序，让模型自己学习多目标之间的最优权衡。

4.4 融合策略的选择建议

场景 推荐策略
目标少（2~3个），业务逻辑清晰 加权融分，Grid Search 找权重
存在明显短板，不容有失 乘积融分，对短板惩罚更强
目标之间存在业务优先级硬约束 帕累托优化 + 分层融合（先保底线，再优化上线）
多业务线统一排序，需自适应 学习融合权重（MLP / LTR）

5. 总结

多目标排序建模的核心挑战在于如何让多个目标在共享参数的同时，保持各自的最优性，常见的解决思路分为三类：

思路 代表算法 核心机制
硬共享 SharedBottom 所有任务强制共享底层参数，简单但容易冲突
软共享（门控） MMoE + Dropout 每个任务独立门控选择 Expert，极化问题是关键瓶颈
显式解耦 PLE 分离共享专家和专属专家，从根本上避免好任务被拖垮
全链路建模 ESMM / ESM2 用链式关系建模多目标，数据稀疏场景下的高效方案
自适应加权 不确定性加权、APTAM 自动学习任务权重，减少人工调参成本

没有"最优"的算法，只有"最适合"的算法——任务相关性高时 SharedBottom 足够用；任务差异大时优先考虑 PLE；CVR 稀疏且存在转化链路时优先 ESMM。实际工程中也可以组合使用，例如 PLE + 不确定性加权 + 乘积融分，以应对复杂的业务场景。

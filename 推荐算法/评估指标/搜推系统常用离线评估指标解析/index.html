搜推系统常用离线评估指标解析

发表于2026-05-20|更新于2026-05-20|推荐算法评估指标

搜推系统离线评估指标详解：从 HitRate、AUC到NDCG指标，手把手拆解核心公式与业务含义。

1. 为什么需要离线评估

线上 A/B 实验是评估算法效果的"金标准"，但它的代价也很高——需要切分流量、长周期观察，另外万一模型翻车还直接影响用户体验。所以在实际迭代中，会先用离线指标快速筛选，再用 A/B 实验做最终验证。

离线评估的指标包括 召回类、预估类、排序类等，覆盖了从候选池到精排的完整 pipeline。需要注意的是，离线指标与线上指标之间存在天然 Gap，离线指标和线上观测指标本身就不一致，也会有曝光偏差和位置偏差等问题，很可能出现离线指标涨，线上指标跌的情况，因此也不能太迷信离线指标。

2. 召回类指标

推荐系统的第一步是从海量候选池中召回候选集，召回类指标衡量的是"有没有把用户感兴趣的内容找出来"。

2.1 Recall@K（召回率 / 覆盖率）

Recall@K 衡量 Top-K 推荐列表对所有正样本的覆盖程度：

Recall@K=Top-K 中命中的正样本数全部正样本数\text{Recall@K} = \frac{\text{Top-K 中命中的正样本数}}{\text{全部正样本数}} Recall@K=全部正样本数Top-K 中命中的正样本数​

计算示例：某用户 uuu 的全部正样本为 {A,B,C,D,E,F}\{A, B, C, D, E, F\}{A,B,C,D,E,F}（共 6 个），模型为该用户输出的 Top-5 推荐列表为 [A,C,E,X,Y][A, C, E, X, Y][A,C,E,X,Y]，其中正样本为 {A,C,E}\{A, C, E\}{A,C,E}（命中 3 个）。

Recall@5(u)=36=0.5\text{Recall@5}(u) = \frac{3}{6} = 0.5 Recall@5(u)=63​=0.5

在实际评估中，通常对所有用户取平均，得到整体的 Recall@K。

2.2 HitRate@K（HR@K，存在性命中）

HitRate@K 衡量 Top-K 推荐列表中是否至少命中一个正样本：

HitRate@K=#{用户∣Top-K 中至少有一个正样本}#{总用户}\text{HitRate@K} = \frac{\#\{\text{用户} \mid \text{Top-K 中至少有一个正样本}\}}{\#\{\text{总用户}\}} HitRate@K=#{总用户}#{用户∣Top-K 中至少有一个正样本}​

计算示例：对 4 个用户进行评估，K=5：

用户 正样本总数 Top-5 推荐 是否命中 Recall@5
u1 6 [A, C, E, X, Y] ✅ 命中（A/C/E） 3/6
u2 2 [M, N, O, P, Q] ❌ 未命中 0/2
u3 4 [D, F, G, H, I] ✅ 命中（D/F） 2/4
u4 3 [J, K, L, M, N] ✅ 命中（J/K） 2/3

HitRate@5=34=75%\text{HitRate@5} = \frac{3}{4} = 75\% HitRate@5=43​=75%

3. 预估类指标

预估类指标衡量模型输出的分数质量，是评估排序模型的打分能力。

3.1 AUC

AUC（Area Under the ROC Curve）的含义是指ROC曲线下的面积，在搜推系统更多表示正样本排在负样本前面的概率，这二者的含义应该是可以证明是等价的。AUC是推荐/广告系统最核心的离线排序模型评估指标，衡量模型区分正负样本的能力。

AUC 的计算

AUC 有三种常见的计算方式，从原理到工程实现逐一说明。

方式一：枚举正负样本对（定义式）

最直接的计算方式是枚举所有正负样本对，统计正样本得分大于负样本的对数，再除以总对数：

AUC=1N+⋅N−∑i=1N+∑j=1N−I(scorei+>scorej−)\text{AUC} = \frac{1}{N_+ \cdot N_-} \sum_{i=1}^{N_+} \sum_{j=1}^{N_-} \mathbb{I}(\text{score}_i^+ > \text{score}_j^-) AUC=N+​⋅N−​1​i=1∑N+​​j=1∑N−​​I(scorei+​>scorej−​)

其中 N+N_+N+​ 和 N−N_-N−​ 分别是正负样本数，I\mathbb{I}I 是指示函数（正样本得分严格大于负样本时为 1，否则为 0）。这恰好就是 AUC 的概率意义：随机抽取一对正负样本，正样本得分更高的概率。

时间复杂度 O(N+⋅N−)O(N_+ \cdot N_-)O(N+​⋅N−​)，但当样本量达到千万级别时直接枚举代价过高。

方式二：排序法（工程主流）

排序法利用样本的全局排序位置来计算 AUC，只需要一次排序，时间复杂度 O(Nlog⁡N)O(N \log N)O(NlogN)，是工程实践中真正使用的方法。

将所有样本（正样本 + 负样本）按预测得分从高到低排序。设正样本数为 N+N_+N+​，负样本数为 N−N_-N−​，正样本在排序序列中的排名（从 0 开始）为 ranki\text{rank}_iranki​（i=1,2,…,N+i = 1, 2, \ldots, N_+i=1,2,…,N+​）。

排序法公式为：

AUC=∑i=1N+ranki−N+(N+−1)2N+⋅N−\text{AUC} = \frac{\sum_{i=1}^{N_+} \text{rank}_i - \frac{N_+ (N_+ - 1)}{2}}{N_+ \cdot N_-} AUC=N+​⋅N−​∑i=1N+​​ranki​−2N+​(N+​−1)​​

其中，∑i=1N+ranki\sum_{i=1}^{N_+} \text{rank}_i∑i=1N+​​ranki​ 是所有正样本排名之和，表示排在正样本前面的样本数量，N+(N+−1)2\frac{N_+ (N_+ - 1)}{2}2N+​(N+​−1)​ 是正样本排在正样本前面的样本数量，相减即可得到排在正样本前面的负样本数量，再除以总对数 N+⋅N−N_+ \cdot N_-N+​⋅N−​ 就得到 AUC。

排序法计算示例

假设有 6 个样本，真实标签和预测得分如下：

样本 标签 预测得分 排序后位置（rank）
A +1 0.9 5（最高）
B +1 0.8 4
C -1 0.7 3
D -1 0.6 2
E -1 0.4 1
F -1 0.2 0（最低）

其中正样本 (A,B)(A, B)(A,B) 排名为 (5,4)(5, 4)(5,4)，负样本 (C,D,E,F)(C, D, E, F)(C,D,E,F) 排名为 (3,2,1,0)(3, 2, 1, 0)(3,2,1,0)。

∑ranki=5+4=9\sum \text{rank}_i = 5 + 4 = 9 ∑ranki​=5+4=9

AUC=9−2×122×4=9−18=88=1.0\text{AUC} = \frac{9 - \frac{2 \times 1}{2}}{2 \times 4} = \frac{9 - 1}{8} = \frac{8}{8} = 1.0 AUC=2×49−22×1​​=89−1​=88​=1.0

模型完美排序，正样本全部排在负样本前面，AUC = 1.0。

再看一个不完美的例子：

样本 标签 预测得分 排序后位置（rank）
A +1 0.9 5
B -1 0.8 4
C -1 0.7 3
D +1 0.6 2
E -1 0.4 1
F -1 0.2 0

正样本 (A,D)(A, D)(A,D) 排名为 (5,2)(5, 2)(5,2)：

∑ranki=5+2=7\sum \text{rank}_i = 5 + 2 = 7 ∑ranki​=5+2=7

AUC=7−2×122×4=7−18=68=0.75\text{AUC} = \frac{7 - \frac{2 \times 1}{2}}{2 \times 4} = \frac{7 - 1}{8} = \frac{6}{8} = 0.75 AUC=2×47−22×1​​=87−1​=86​=0.75

方式三：梯形积分法

逐步调整分类阈值，计算每个阈值下对应的 (FPR,TPR)(\text{FPR}, \text{TPR})(FPR,TPR) 点，然后用梯形法则对 ROC 曲线积分。这种方式常用于绘制 ROC 曲线本身，AUC 只是积分的结果。

从混淆矩阵到 AUC

AUC除了表示模型的排序能力，还可以通过混淆矩阵中的 TPR（True Positive Rate）和 FPR（False Positive Rate）来理解。给定一个分类阈值，可以得到混淆矩阵：

预测=1 预测=0
真实=1 TP FN
真实=0 FP TN

由此定义两个关键比率：

TPR（召回率）=TPTP+FN,FPR=FPFP+TN\text{TPR（召回率）} = \frac{TP}{TP + FN}, \quad \text{FPR} = \frac{FP}{FP + TN} TPR（召回率）=TP+FNTP​,FPR=FP+TNFP​

不断调整阈值，就得到一系列 (FPR,TPR)(\text{FPR}, \text{TPR})(FPR,TPR) 点，连成的曲线就是 ROC 曲线，曲线下的面积就是 AUC。

AUC 的取值范围是 [0,1][0, 1][0,1]，含义如下：

AUC 含义
0.5 随机模型，没有区分能力
0.6–0.9 有一定区分能力
>0.9 很强，需检查是否有标签泄露风险

为什么 AUC 不随阈值变化？

这是 AUC 最重要的性质：它衡量的是模型的排序能力，而非某个阈值下的分类准确率。

AUC 的概率意义是：随机抽取一个正样本和负样本，正样本得分高于负样本的概率。即：

AUC=P(score正>score负)\text{AUC} = P(\text{score}_{\text{正}} > \text{score}_{\text{负}}) AUC=P(score正​>score负​)

这个概率只和样本得分的相对排序有关，和阈值无关。所以无论把分类阈值设在 0.5 还是 0.9，AUC 值始终不变。

举例：正样本得分 [0.9, 0.8, 0.6]，负样本得分 [0.4, 0.3, 0.2]，正样本得分全部高于负样本 → 模型完美排序，AUC = 1.0。

3.2 GAUC —— 分组 AUC

AUC 衡量的是全局排序能力，但在推荐系统中，不同用户的打分基准不同——有的用户活跃、打分偏高，有的用户沉默、打分偏低。直接用全局 AUC 对比两个模型，可能掩盖用户维度的真实差异。

GAUC（Group AUC）将 AUC 计算限制在每个用户（组）内部：

GAUC=1∑w∑uwu⋅AUCu\text{GAUC} = \frac{1}{\sum_w} \sum_{u} w_u \cdot \text{AUC}_u GAUC=∑w​1​u∑​wu​⋅AUCu​

其中 wuw_uwu​ 是用户 uuu 的曝光权重，AUCu\text{AUC}_uAUCu​ 是用户 uuu 内部的 AUC。实际工程中通常用曝光次数或点击次数作为权重。

边界情况处理

GAUC 按用户分组计算时，常遇到两类边界情况：

情况一：用户没有正样本（全是负样本）

这是最常见的边界情况，该用户组内无法计算 AUC。常见处理策略：

处理策略 做法
跳过（最常用） 只对有正样本的用户计算 GAUC
赋值为 0.5 视为"无法评估"，对均值无偏
赋值为 0 悲观假设排序最差
赋值为 1 乐观假设排序最优

GAUC=∑u:N+u>0wu⋅AUCusumu:N+u>0wu\text{GAUC} = \frac{\sum_{u: N_+^u > 0} w_u \cdot \text{AUC}_u}{sum_{u: N_+^u > 0} w_u} GAUC=sumu:N+u​>0​wu​∑u:N+u​>0​wu​⋅AUCu​​

情况二：用户没有负样本（全是正样本）

这种情况较少见，通常发生在用户行为极为密集的场景。此时：

  • 如果用户只有正样本，没有负样本作为对比，AUC 定义模糊——无法判断模型是否"排出"了好样本
  • 常见处理：跳过该用户，或设置 AUCu=1.0\text{AUC}_u = 1.0AUCu​=1.0（默认排序最优，但需谨慎）
场景 建议处理
少数异常用户（极稀疏/极密集） 直接跳过，不参与 GAUC 计算
规模化处理 设置阈值：正样本数 ≥2\geq 2≥2 且负样本数 ≥2\geq 2≥2 才参与计算

3.3 PCOC —— 概率校准系数

PCOC（Predicted CTR / Observed CTR）衡量预估点击率是否和真实点击率吻合：

PCOC=Average Predicted CTRAverage Observed CTR\text{PCOC} = \frac{\text{Average Predicted CTR}}{\text{Average Observed CTR}} PCOC=Average Observed CTRAverage Predicted CTR​

PCOC 值 含义
PCOC = 1.0 完美校准，预估分和真实分布完全一致
PCOC > 1.0 高估，模型报的点击率比实际高
PCOC < 1.0 低估，模型报的点击率比实际低

举例：线上 1000 次曝光，实际产生了 50 次点击，Observed CTR = 5%；模型对这 1000 次曝光的 Predicted CTR 平均值为 6%，则 PCOC = 6% / 5% = 1.2，模型高估了约 20%。

PCOC 在广告计费（按点击收费）和出价系统中至关重要：高估会导致出价过高、浪费预算，低估会导致出价过低、拿不到量。此外，PCOC 也常作为模型 calibration（校准）层的目标。

4. 排序类指标

排序类指标衡量Top-K 列表的排序质量

4.1 NDCG@K —— 核心排序指标

NDCG（Normalized Discounted Cumulative Gain）是推荐系统最常用的排序指标，它的设计哲学是：位置越靠前，命中收益越大。

CG → DCG → NDCG 的递进

**CG（累积收益）**是最简单的版本，不考虑位置：

CG@K=∑i=1Kreli\text{CG@K} = \sum_{i=1}^{K} \text{rel}_i CG@K=i=1∑K​reli​

其中 reli\text{rel}_ireli​ 是第 iii 位的相关性分数（可以是二值1/0，比如用户有行为/无行为；也可以是多值0/1/2,…，比如无行为0、点击1、收藏2、关注3等）。

DCG（折扣累积收益）引入位置折扣因子，越靠后的位置收益越小。同时，DCG 使用指数增益函数 2reli−12^{\text{rel}_i} - 12reli​−1，使得高相关性 item 的收益远高于低相关性 item：

DCG@K=∑i=1K2reli−1log⁡2(i+1)\text{DCG@K} = \sum_{i=1}^{K} \frac{2^{\text{rel}_i} - 1}{\log_2(i+1)} DCG@K=i=1∑K​log2​(i+1)2reli​−1​

对数折扣的直觉：用户注意力随位置递减，前几名之间的差异远比后几名重要。指数增益的直觉：高相关 item 的价值远高于低相关 item，应该被更强烈地"奖励"。

注：若 reli∈{0,1}\text{rel}_i \in \{0, 1\}reli​∈{0,1}（二值相关），则 2reli−1∈{0,1}2^{\text{rel}_i} - 1 \in \{0, 1\}2reli​−1∈{0,1}，简化为 relilog⁡2(i+1)\frac{\text{rel}_i}{\log_2(i+1)}log2​(i+1)reli​​。

NDCG（归一化 DCG） 用理想排序的 DCG 做归一化，使得不同长度的列表可比：

NDCG@K=DCG@KIDCG@K\text{NDCG@K} = \frac{\text{DCG@K}}{\text{IDCG@K}} NDCG@K=IDCG@KDCG@K​

其中 IDCG@K（Ideal DCG）是按相关性降序排列的最优结果。NDCG 的取值范围是 [0,1][0, 1][0,1]，1 表示完美排序。

NDCG 计算示例

假设有 6 个 item，预测得分和真实相关度如下，rel 值分别对应 gain = 2rel−12^{\text{rel}} - 12rel−1：

item 预测得分 真实 rel gain = 2rel−12^{\text{rel}} - 12rel−1
A 0.95 2 3
B 0.80 0 0
C 0.70 2 3
D 0.50 1 1
E 0.40 2 3
F 0.30 0 0

按预测得分排序得到推荐列表 [A, B, C, D, E, F]，对应的 gain 序列为 [3, 0, 3, 1, 3, 0]。

计算 DCG@6：

DCG@6=3log⁡22+0log⁡23+3log⁡24+1log⁡25+3log⁡26+0log⁡27=3+0+1.5+0.43+1.15+0≈6.08\text{DCG@6} = \frac{3}{\log_2 2} + \frac{0}{\log_2 3} + \frac{3}{\log_2 4} + \frac{1}{\log_2 5} + \frac{3}{\log_2 6} + \frac{0}{\log_2 7} = 3 + 0 + 1.5 + 0.43 + 1.15 + 0 \approx 6.08 DCG@6=log2​23​+log2​30​+log2​43​+log2​51​+log2​63​+log2​70​=3+0+1.5+0.43+1.15+0≈6.08

IDCG@6=3log⁡22+3log⁡23+3log⁡24+1log⁡25+0log⁡26+0log⁡27=3+1.89+1.5+0.43+0+0≈6.82\text{IDCG@6} = \frac{3}{\log_2 2} + \frac{3}{\log_2 3} + \frac{3}{\log_2 4} + \frac{1}{\log_2 5} + \frac{0}{\log_2 6} + \frac{0}{\log_2 7} = 3 + 1.89 + 1.5 + 0.43 + 0 + 0 \approx 6.82 IDCG@6=log2​23​+log2​33​+log2​43​+log2​51​+log2​60​+log2​70​=3+1.89+1.5+0.43+0+0≈6.82

NDCG@6=6.086.82≈0.89\text{NDCG@6} = \frac{6.08}{6.82} \approx 0.89 NDCG@6=6.826.08​≈0.89

4.2 MRR@K 与 MAP@K

MRR（Mean Reciprocal Rank）和 MAP（Mean Average Precision）同样是排序指标，但它们更适合搜索、问答等精确匹配优先的场景，而非个性化推荐。

MRR@K（平均倒数排名）

MRR 只看第一个命中的位置：

MRR@K=1Q∑q=1Q1rankq\text{MRR@K} = \frac{1}{Q} \sum_{q=1}^{Q} \frac{1}{\text{rank}_q} MRR@K=Q1​q=1∑Q​rankq​1​

其中 rankq\text{rank}_qrankq​ 是查询 qqq 的第一个相关结果的位置。

举例：查询"推荐系统评估指标"，相关结果排在第 3 位 → MRR = 1/3 ≈ 0.333。

MRR 的特点是简单直接，适合"用户只点第一个结果"或"只要有一个对就行"的场景。

MAP@K（平均精度均值）

MAP 同时考虑命中个数和各自位置：

AP@K=1min⁡(m,K)∑i=1KI(reli>0)⋅#{前 i 位中的正样本}i\text{AP@K} = \frac{1}{\min(m, K)} \sum_{i=1}^{K} \mathbb{I}(\text{rel}_i > 0) \cdot \frac{\#\{\text{前 } i \text{ 位中的正样本}\}}{i} AP@K=min(m,K)1​i=1∑K​I(reli​>0)⋅i#{前 i 位中的正样本}​

其中 mmm 是总正样本数，AP 对每个命中位置计算"截止到该位置为止命中了多少个"。

MAP@K=1Q∑q=1QAP@Kq\text{MAP@K} = \frac{1}{Q} \sum_{q=1}^{Q} \text{AP@K}_q MAP@K=Q1​q=1∑Q​AP@Kq​

5. 总结

类别 指标 核心含义
召回类 Recall@K 覆盖率：Top-K 找出了多少比例的正样本
召回类 HitRate@K 存在性：Top-K 中有没有至少命中一个
预估类 AUC 模型区分正负样本的排序能力
预估类 GAUC 用户粒度的 AUC，解决用户间可比性问题
预估类 PCOC 概率校准：预估 CTR 与真实 CTR 的吻合程度
排序类 NDCG@K 归一化折扣累积收益，考虑位置价值
排序类 MRR@K / MAP@K 第一个命中位置 / 精度均值，适合搜索/问答场景

离线评估是算法迭代的加速器，核心是用对指标、走对方向。离线指标虽然不能完全替代线上 A/B 实验，但它们是快速验证和迭代的利器，能帮助我们在海量候选方案中迅速找到潜力股，节省宝贵的线上验证资源。

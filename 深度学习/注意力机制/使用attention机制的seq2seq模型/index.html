使用attention机制的seq2seq模型

发表于2026-03-08|更新于2026-03-08|深度学习注意力机制

基于RNN的Seq2Seq模型通过RNN编码器（Encoder）将整个输入序列压缩成一个固定长度的向量（通常是最后一个时间步的隐藏状态 hidden state），然后RNN解码器（Decoder）基于该状态向量生成翻译结果。

存在问题：

  • 在该模型结构下，输入句子较长时，仅依赖编码器向量很难容纳所有信息，导致长距离依赖丢失，翻译质量会严重下降。
  • Seq2Seq是只用了最后一个时间步的输出，但在翻译时，希望在翻译某个词时能够关注原句子中对应的词。比如翻译“hello，world”成“你好，世界”时，“你好”想要更关注“hello”，而这在Seq2Seq未做建模

故尝试将注意力机制引入到seq2seq模型中，以解决仅靠单个编码向量无法准确建模的问题。

  • 在编码器阶段输出每个时间步的隐藏状态集合：H=(h1,h2,h3,...,hT)H=(h_1, h_2, h_3,...,h_T)H=(h1​,h2​,h3​,...,hT​)
  • 在解码器阶段预测第ttt个输出yty_tyt​时
    • 利用解码器上一时刻的状态st−1s_{t-1}st−1​（作为 Query）和编码器的所有隐藏状态HHH（作为 Keys和Values），计算注意力 attention=at=∑ia(st−1,hi)hi\text{attention} = a_t = \sum_i a(s_{t-1}, h_i)h_iattention=at​=∑i​a(st−1​,hi​)hi​。在seq2seq模型中使用attention机制时，query和key都是RNN模块的隐状态向量，这样能够保持在同一个状态空间。
    • 合并上一个词（t−1t-1t−1）的输出和attention结果，预测时刻ttt的输出yt=RNN(at,st−1)y_t = RNN(a_t, s_{t-1})yt​=RNN(at​,st−1​)

在seq2seq模型中引入注意力机制，是由Bahdanau、bengio等人2014年在论文《Neural Machine Translation by Jointly Learning to Align and Translate》中提出，显著提升了机器翻译任务中长句子的处理效果。随后在2015年，Minh-Thang Luong等人发表了《Effective Approaches to Attention-based Neural Machine Translation》，对注意力机制进行了进一步的简化和改进（提出了Luong Attention），使其更加流行和易于实现。

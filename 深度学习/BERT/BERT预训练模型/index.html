BERT预训练模型

>*BERT 开启 NLP 预训练新时代*

1. BERT 的诞生：动机与历史背景

1.1 论文信息

2018 年，Google 研究团队在论文 《BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding》 中提出了 BERT（Bidirectional Encoder Representations from Transformers） 模型。

缩写 全称 含义
B Bidirectional 双向建模
ER Encoder Representations Transformer 编码器的表示
T from Transformers 基于 Transformer 架构

📄 论文信息：
Devlin et al., “BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding”, NAACL 2019

论文地址：https://arxiv.org/abs/1810.04805

1.2 提出的动机

在 BERT 之前，NLP 领域的主流范式是 词向量（word2vec/GloVe） + 序列建模（LSTM/Seq2Seq）,BERT 的提出是为了解决一个核心问题：

能否训练一个通用的"基座模型"，在大量无标注语料上预训练，学习通用的语言理解能力，然后在各种下游任务上做微调？

这与 Word2Vec 的思路类似——用无监督方式学习通用表示，但 BERT 不只是抽取特征，而是构建了可微调的预训练模型，BERT 开启了 NLP 领域的预训练 + 微调范式——在此之前，每个 NLP 任务都需要从头训练模型；在此之后，只需在大规模预训练模型上微调即可。

2. BERT 的模型结构

2.1 架构基础：基于 Transformer 编码器

BERT 的核心架构是 Transformer 的编码器部分——只使用编码器，不使用解码器。

这与 GPT 的单向生成架构（只用解码器）刚好相反：

模型 架构 注意力方向 典型任务
BERT 仅编码器（Encoder） 双向：看到上下文 理解任务（分类、NER、QA）
GPT 仅解码器（Decoder） 单向：只看前文 生成任务（文本生成）
T5 编码器 + 解码器 混合 Seq2Seq（翻译、摘要）

BERT 只用编码器的核心优势：每个位置都能看到完整上下文，非常适合理解类任务。

2.2 对 Transformer 编码器的改造

在 Transformer 原始编码器的基础上，BERT 做了以下关键改造：

改造方面 原始 Transformer 编码器 BERT 编码器
位置编码 Sin/Cos 固定编码 可学习的位置编码
输入形式 单句或单段 句子对（Sentence Pair）
Embedding Token + Position Token + Position + Segment

（1）位置编码：改为可学习

Transformer 原始论文使用固定的 Sin/Cos 周期函数编码位置。

BERT 改为可学习的位置编码——位置向量作为模型参数，在预训练中自动学习得到。

（2）输入形式：改为句子对

原始 Transformer 的输入通常是单个序列（如一段话）。

BERT 的输入设计为句子对形式，并在输入中加入特殊 token：

特殊 token 作用
[CLS] 放在句首，其输出向量作为整个序列的聚合表示（用于句子级分类）
[SEP] 放在每个句子末尾，标记句子之间的分隔

（3）Segment Embedding：区分句子A和句子B

BERT 引入了第三种 Embedding——Segment Embedding：

Embedding 类型 作用 示例
Token Embedding 词向量 “我” → 768维向量
Position Embedding 位置向量 第1个词 → 可学习的Position #1
Segment Embedding 句子段向量 属于句子A → E_A；属于句子B → E_B

Segment Embedding 让模型区分当前 token 属于哪个句子，为下一句预测（NSP）任务提供支持。

2.3 为什么采用句子对作为输入？

这是理解 BERT 设计意图的关键问题。BERT 在预训练阶段的输入形式根据任务不同而有所区别：

预训练阶段：MLM vs NSP 输入形式不同

预训练任务 输入形式 说明
MLM（掩码语言模型） 单个句子或两个连续句子 随机遮盖 15% 的词元，根据上下文预测被遮盖词
NSP（下一句预测） 必须两个句子 50% 正例（连续句子），50% 负例（随机句子）

MLM 任务：输入可以是单个句子或两个连续的句子。核心是学习词元的双向上下文表示，因此可以是任意长度的文本片段。

NSP 任务：输入必须是两个句子。模型判断第二个句子是否是第一个句子的下一句，50% 正例（连续句子），50% 负例（随机句子）。

💡 关键区分：MLM 的输入可以是单句，NSP 的输入必须是句子对，但一般在训练中，为了和NSP任务保持一致，MLM任务也选择输入句子对。两个任务共用同一套 BERT 编码器，共享参数。

深层原因：下游任务大量需要句子关系理解

NLP 中许多重要任务本质上都是句子关系判断：

下游任务 输入形式 本质
句子匹配 句子A vs 句子B 判断两句话是否相关
问答系统 问题 vs 段落 判断段落是否为答案
自然语言推理 前提 vs 假设 判断逻辑关系
对话 历史对话 vs 当前回复 判断回复是否合适

预训练时使用句子对，能让模型学会理解两个文本之间的关系，从而更好地迁移到下游任务。

微调阶段：输入形式取决于具体任务

下游任务 输入形式
文本分类（单句情感等） 单个句子
句子对分类（自然语言推理） 两个句子
问答系统 问题 + 段落，两个句子

微调阶段完全根据任务需要设计输入，不再受预训练任务形式的限制。

💡 核心总结：句子对设计的目的是：让 BERT 在预训练时就学会理解任意两段文本之间的关系，这使得预训练学到的知识能直接迁移到各种下游任务。

2.4 Base 与 Large 版本

BERT 提供了两个规模的版本：

对比维度 BERT-Base BERT-Large
Transformer 层数 N 12 24
注意力头数 12 16
隐藏层维度 d_model 768 1024
FFN 中间维度 3072 4096
参数量 约 1.1 亿 约 3.4 亿

💡 为什么有两个版本？

BERT-Base 适合在单卡 GPU 上训练和部署，参数量约 1.1 亿；BERT-Large 在大规模数据上效果更好，但训练和推理成本更高，参数量约 3.4 亿。

3. BERT 的预训练任务

BERT 的创新不仅在于输入上的设计，另外还在于两个预训练任务的设计。

3.1 掩码语言模型（Masked Language Model, MLM）

核心思想：完形填空（Cloze Task）

语言模型通常只能从左到右预测下一个词——但 BERT 是双向的，怎么设计训练任务？

MLM 的做法：随机将输入中的一些词替换为 [MASK] token，然后让模型预测被遮盖的词。

示例：

细节：

  • 随机遮盖 15% 的 token
  • 其中 80% 替换为 [MASK]
  • 10% 替换为随机词（增加鲁棒性）
  • 10% 保持不变（减少 [MASK] 在微调时的影响）

** BERT的MLM任务为什么训练采用句子对？ **

MLM 任务本身不需要句子对，只需要有足够上下文来预测被遮盖的词。

但 BERT 在训练时用句子对，是因为另一个预训练任务 NSP 需要句子对——为了两个任务联合训练，BERT 将所有输入都设计为句子对格式。

推理时：MLM 任务对输入格式没有限制，不需要句子对——只需按照具体任务设计输入即可。

3.2 下一句预测（Next Sentence Prediction, NSP）

核心思想：判断句子 B 是否是句子 A 的下一句

BERT 被设计用于问答、推理等需要理解句子关系的任务，因此增加了 NSP 任务。

正样本：句子 A 的下一句 → IsNext = True

负样本：从语料中随机选取非相邻句子 → IsNext = False

3.3 两个任务的联合训练

BERT 的预训练是同时优化 MLM 和 NSP 两个任务：

  • MLM Loss：交叉熵损失，预测被遮盖的词
  • NSP Loss：二分类交叉熵损失，判断是否相邻

两个任务共享同一个 Transformer 编码器，编码器参数在两个任务上联合优化。

4. BERT的微调与使用

4.1 为什么需要微调？

BERT 的预训练任务（完形填空、下一句判断）在实际 NLP 任务中很少直接用到。

BERT 的真正价值在于：学习到了通用的语言理解表示，可以在下游任务上微调。

微调（Fine-tuning）：在预训练 BERT 的基础上，添加一个任务特定的输出层，然后在下游任务的标注数据上继续训练。

4.2 BERT 微调实战

任务一：句子分类（Sentiment Classification）

要素 说明
输入 单个句子（如情感评论）
关键 使用 [CLS] token 的输出向量
输出层 分类器（Softmax）→ 正面 / 负面

任务二：命名实体识别（NER）

要素 说明
输入 句子中每个词
关键 使用每个词对应的输出向量
输出层 序列标注器 → PER / ORG / LOC / O

任务三：问答系统（Question Answering）

要素 说明
输入 问题 + 段落（用 [SEP] 分隔）
关键 预测答案在段落中的起止位置
输出层 两个分类器（Start Position / End Position）

任务四：自然语言推理（NLI）

要素 说明
输入 前提 + 假设（用 [SEP] 分隔）
输出层 三分类器 → 蕴含 / 矛盾 / 中立

4.3 微调的核心要点

要点 说明
学习率 预训练使用 2e-5 ~ 5e-5（比从头训练低很多）
训练轮数 通常 3~5 个 epoch
batch size 16~32（BERT-Base）
冻结策略 通常全部参数微调（也可冻结底层）

💡 为什么微调学习率要这么小？

预训练模型已经学到了大量通用知识，过大的学习率会破坏这些知识，导致模型"遗忘"。通常比从头训练小 10~100 倍。

5. BERT 的影响与后续发展

5.1 BERT 开启了预训练时代

BERT 的意义不仅在于刷新了各项 NLP 基准测试，更重要的是确立了预训练 + 微调的范式：

  1. 大规模无监督预训练：利用海量无标注文本学习通用语言表示
  2. 任务特定微调：在下游任务少量标注数据上微调
  3. 知识迁移：预训练中学到的知识可以迁移到各种下游任务

5.2 BERT 的后续改进

模型 改进方向 关键创新
RoBERTa 训练策略优化 去掉 NSP，动态掩码，更大数据
ALBERT 参数效率 共享层参数，因子分解Embedding
ELECTRA 训练效率 替换token检测替代MLM
DistilBERT 模型蒸馏 知识蒸馏压缩 60% 参数
Sentence-BERT 语义相似度 学习句子级别表示
BioBERT 领域适应 医学领域预训练
Chinese-BERT 中文优化 全词掩码，中文预训练

5.3 BERT 与 GPT 的对比

维度 BERT GPT
架构 仅编码器 仅解码器
注意力 双向 单向（只看前文）
训练任务 MLM + NSP 下一个 token 预测
优势任务 理解任务 生成任务
典型应用 分类、NER、QA 文本生成、对话

结语

BERT 的核心贡献可以总结为三点：

  1. 双向建模：利用 Transformer 编码器的双向注意力，同时捕获上下文信息
  2. 预训练 + 微调范式：大规模无监督预训练 + 少量标注数据微调，大幅降低 NLP 任务的数据门槛
  3. 两个训练任务：MLM（掩码语言模型）+ NSP（下一句预测），共同学习语言理解和句子关系

BERT 开启了 NLP 领域的大模型时代。后续的 GPT 系列（GPT-3/4）、LLaMA、ChatGPT 等，都在 BERT 开创的范式上继续演进，最终催生了今天的大语言模型浪潮。

transformer中的FFN详解

发表于2026-06-06|更新于2026-06-06|深度学习Transformer

Transformer 里的 FFN，本质上是一个 MLP。区别不在公式，而在使用方式：它对每个 token 位置单独执行，并且所有位置共享同一组参数。从这个角度看，它也等价于两层 kernel size 为 1 的卷积。

1. FFN 本质上就是 MLP

Transformer block 里除了 Attention 子层，还会接一个 Feed-Forward Network，也就是 FFN。

从公式看，Transformer FFN 通常就是一个两层 MLP：

FFN⁡(xi)=W2 σ(W1xi+b1)+b2\operatorname{FFN}(x_i) = W_2 \, \sigma(W_1 x_i + b_1) + b_2 FFN(xi​)=W2​σ(W1​xi​+b1​)+b2​

这里 xix_ixi​ 是第 iii 个 token 的隐藏向量，σ\sigmaσ 是激活函数，比如 ReLU、GeLU、SwiGLU 等。

FFN 的第一层通常把维度从 dmodeld_{\text{model}}dmodel​ 扩到 dffd_{\text{ff}}dff​，第二层再投影回 dmodeld_{\text{model}}dmodel​：

W1∈Rdff×dmodelW_1 \in \mathbb{R}^{d_{\text{ff}} \times d_{\text{model}}} W1​∈Rdff​×dmodel​

W2∈Rdmodel×dffW_2 \in \mathbb{R}^{d_{\text{model}} \times d_{\text{ff}}} W2​∈Rdmodel​×dff​

所以，单看这一行公式，Transformer FFN 和普通 MLP 没有本质区别。真正的差异在于：普通 MLP 通常处理一个整体向量，而 Transformer FFN 是对序列里的每个位置分别处理。

2. 普通 MLP 和 Transformer FFN 的差异

普通 MLP，也就是 Multi-Layer Perceptron，通常被看成一个向量到另一个向量的非线性映射：

y=W2 σ(W1x+b1)+b2y = W_2 \, \sigma(W_1 x + b_1) + b_2 y=W2​σ(W1​x+b1​)+b2​

它可以用于分类头、推荐系统打分、表格特征建模、embedding 拼接后的特征交叉等。大多数情况下，输入 xxx 是一个完整的 feature vector。

比如在推荐系统里，可以把用户年龄、城市、历史行为 embedding、物品类别 embedding 拼到一起，再送进 MLP。这个 MLP 处理的是“这一条样本的整体特征”。

如果你把一句话里所有 token 的 embedding 展平：

x=[x1;x2;⋯ ;xL]x = [x_1; x_2; \cdots; x_L] x=[x1​;x2​;⋯;xL​]

再送进一个 MLP，那么第一层权重可以同时连接不同 token 的维度。这样的 MLP 可以直接混合不同位置的信息。

Transformer FFN 不是这么做的。它不会把整个序列展平成一个大向量，而是把同一个小 MLP 应用到每个 token 位置上：

yi=FFN⁡(xi)y_i = \operatorname{FFN}(x_i) yi​=FFN(xi​)

也就是说，在 FFN 内部，yiy_iyi​ 只由当前位置的 xix_ixi​ 计算得到，不会直接读取 xjx_jxj​。如果 xix_ixi​ 里已经有其他 token 的信息，那通常是前面的 Self-Attention 已经把上下文聚合进来了。

这个差异可以概括成一句话：Transformer FFN 默认不混合位置，只混合同一 token 内部的特征维度。

3. position-wise：逐位置共享参数

Transformer 原论文把 FFN 称为 position-wise feed-forward network。position-wise 指的是：每个位置单独计算，但所有位置共用同一套 FFN 参数。

设某层 FFN 的输入是：

X∈RB×L×dmodelX \in \mathbb{R}^{B \times L \times d_{\text{model}}} X∈RB×L×dmodel​

这里的 XXX 是送入 FFN 子层的张量。它通常来自同一个 Transformer block 里的 Attention 子层及其残差、归一化之后。

把一个样本里的序列写成：

X=[x1,x2,⋯ ,xL]X = [x_1, x_2, \cdots, x_L] X=[x1​,x2​,⋯,xL​]

FFN 的输出是：

Y=[y1,y2,⋯ ,yL]Y = [y_1, y_2, \cdots, y_L] Y=[y1​,y2​,⋯,yL​]

其中每个位置独立计算：

yi=FFN⁡(xi)y_i = \operatorname{FFN}(x_i) yi​=FFN(xi​)

展开后就是：

y1=FFN⁡(x1),y2=FFN⁡(x2),⋯ ,yL=FFN⁡(xL)y_1 = \operatorname{FFN}(x_1), \quad y_2 = \operatorname{FFN}(x_2), \quad \cdots, \quad y_L = \operatorname{FFN}(x_L) y1​=FFN(x1​),y2​=FFN(x2​),⋯,yL​=FFN(xL​)

Self-Attention 负责建模位置之间的关系，让每个 token 从其他 token 处聚合上下文信息；FFN 只对当前位置已经得到的表示做非线性加工。

这也是 FFN 能自然处理变长序列的原因之一。它只依赖最后一维的大小，不依赖这句话有多少个 token。

4. 为什么 FFN 等价于 1x1 卷积

1x1 卷积只看当前空间位置，不看邻居。它做的事情是：在每个位置上，对通道维做一次线性组合，并且所有位置共享同一组卷积核参数。

这和 FFN 的 Linear(d_model, d_ff) 是同一类操作：每个 token 位置独立计算，只混合当前位置的通道维。

5. 总结

Transformer FFN 和普通 MLP 的关系可以这样记：

对比点 普通 MLP Transformer FFN
数学形式 多层线性层加激活函数 通常是两层 MLP，也可能是门控 MLP
输入对象 一个整体特征向量 每个 token 的 hidden state
是否跨 token 混合 取决于输入是否 flatten 或拼接 FFN 内部不直接跨 token 混合
参数共享 对 batch 中样本共享 对 batch 和 sequence 中所有位置共享
输入输出形状 由任务决定 通常 [B, L, d_model] -> [B, L, d_model]
与卷积关系 不一定有卷积解释 等价于两层 kernel_size=1 的 Conv1d

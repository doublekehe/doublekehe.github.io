用vscode解决md文章图片路径问题

发表于2026-03-01|更新于2026-03-01|代码编程markdown

在hexo框架下写markdown文章时，一般无法直接粘贴图片到文章中，需要写图片路径，比如图片url网址，或者是先把图片存在某个路径下，再用图片路径来展示图片。这带来诸多不便，可以尝试用vscode中的插件来解决这个问题。

开启hexo资源文件夹

在hexo主目录下的config.yml配置文件中开启文章的资源文件夹，以存放文章对应的资源文件

这样在新建文章时，就会默认创建一个对应的文件夹，比如 hexo new post “逆波兰表达式”，就会同时创建对应的文件夹如下：

配置paste image插件

Visual Studio Code（VSCode）中使用 Paste Image 插件可以极大提升 Markdown 文档写作时插入图片的效率。该插件支持将剪贴板中的截图一键保存为本地文件，并自动在 Markdown 中插入符合语法规范的相对路径引用，避免了手动保存、命名和编写路径的繁琐操作。想要正确的实现插入图片的功能，需要在paste image插件做相关配置，

打开对应的插件配置页如下：

图片保存路径（Paste Image: path）

这是最关键的配置项，用于指定复制图片保存的位置。推荐采用相对于当前 Markdown 文件的路径结构，以增强项目可移植性。

  • ${currentFileDir}：当前 Markdown 文件所在目录；
  • ${currentFileNameWithoutExt}：当前文件名（不含 .md 后缀）；

开启粘贴图片确认框

开启粘贴图片确认框后，每次粘贴图片，都会弹出对话框，可以修改目录和图片名称。

粘贴图片cmd + opt + v

这里需要注意，需要使用 cmd + opt + v来复制粘贴，才能调用 paste image 插件功能。直接用 control + v无法启用插件。

安装 hexo-asset-img插件

vs code上的配置完成后，在vs code上编辑文章时，图片可以正常粘贴保存，也可以预览文章。但在hexo上生成网页后，无法正常解析路径，需要用其它方式来将markdown文章中的相对图片路径转换成对应的网址图片地址。
yiyungent开发的hexo-asset-img插件可以实现该功能。当在 Markdown 文章中写 ![](image.jpg) 时，该插件会自动将图片路径转换为正确的绝对 URL（例如 /2024/04/18/post-name/image.jpg），确保图片在部署后能正常显示。
详见作者的github地址：https://github.com/yiyungent/hexo-asset-img

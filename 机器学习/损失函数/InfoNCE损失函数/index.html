InfoNCE损失函数

发表于2026-05-31|更新于2026-05-31|机器学习损失函数

InfoNCE 是对比学习的核心目标函数，本质上是在最大化查询与正样本之间的互信息。理解它的公式、温度参数以及与互信息的关系，是深入掌握对比学习的关键。

1. 对比学习与 InfoNCE 的引入

深度学习强大，但长期依赖海量标注数据。无监督学习是打破这一瓶颈的关键，而对比学习（Contrastive Learning）是其中最成功的路线之一。

对比学习的思想极为直观：让相似样本在特征空间中靠得更近，让不相似样本离得更远。这个"相似"由任务定义——可以是同一张图片的不同增广视图，可以是同一语音片段的不同编码，也可以是上下文中相邻的词 token。

2018 年，Oord 等人在论文 Contrastive Predictive Coding（CPC）[^1] 中提出了 InfoNCE 损失函数，迅速成为对比学习领域的标准目标函数。它不仅用于 CPC 本身，还被广泛借鉴到其它一系列相关工作中。

2. InfoNCE 公式与直观理解

2.1 标准形式

InfoNCE 损失函数的核心形式如下：

LInfoNCE=−E[log⁡exp⁡(s(zt,ct)/τ)∑i=1Kexp⁡(s(zt,ci)/τ)]\mathcal{L}_{\text{InfoNCE}} = -\mathbb{E}\left[\log \frac{\exp\left(s\left(z_t, c_t\right) / \tau\right)}{\sum_{i=1}^{K} \exp\left(s\left(z_t, c_i\right) / \tau\right)}\right] LInfoNCE​=−E[log∑i=1K​exp(s(zt​,ci​)/τ)exp(s(zt​,ct​)/τ)​]

其中：

  • ztz_tzt​ 是查询（query）样本的表示向量
  • ctc_tct​ 是与 ztz_tzt​ 对应的正样本（positive）
  • cic_ici​（i≠ti \neq ti=t）是负样本（negative），来自batch内的负样本
  • s(⋅,⋅)s(\cdot, \cdot)s(⋅,⋅) 是相似度函数，通常取 cosine相似度
  • τ\tauτ 是温度系数（temperature），控制分布的平滑程度

分子的作用是让查询与其正样本的相似度越高越好；分母则将正样本与所有负样本进行比较——目标是最小化分母，即让正样本的相对相似度在所有样本中脱颖而出。

2.2 为什么叫"NCE"

NCE 全称 Noise Contrastive Estimation（噪声对比估计），最初由 Gutmann 和 Hyvärinen 提出[^2]。它的核心洞察是：当真实数据分布难以直接计算时，可以将问题转化为一个二分类——哪些样本来自真实分布，哪些是噪声（负样本）。InfoNCE 将这一思想引入到表示学习中，用负样本作为"噪声"来对比学习。

2.3 temperature 的影响

temperature τ\tauτ 决定了 softmax 输出的平滑程度：

  • τ\tauτ 较小（如 0.1）：分布变得尖锐，模型更关注区分最难的负样本——学到更细粒度的特征，但容易陷入局部最优
  • τ\tauτ 较大（如 1.0）：分布更平滑，模型关注整体对比信号——训练更稳定，但区分度可能不足

实践中，τ\tauτ 通常取 0.07～0.2 之间，是对比学习最重要的超参之一。

3. 为什么 InfoNCE 有效——互信息视角

理解 InfoNCE 最深刻的角度是：它本质上是在最大化两个变量之间的互信息。

3.1 互信息的定义

两个随机变量 XXX 和 YYY 的互信息定义为：

I(X;Y)=Ep(x,y)[log⁡p(x∣y)p(x)]=DKL(p(x,y)∥p(x)p(y))I(X; Y) = \mathbb{E}_{p(x,y)}\left[\log \frac{p(x|y)}{p(x)}\right] = D_{\text{KL}}\left(p(x,y) \| p(x)p(y)\right) I(X;Y)=Ep(x,y)​[logp(x)p(x∣y)​]=DKL​(p(x,y)∥p(x)p(y))

互信息衡量的是：知道 YYY 之后，对 XXX 的不确定性的减少程度。对比学习中，我们希望查询 ztz_tzt​ 知道的内容越多关于正样本 ctc_tct​ 的信息。

3.2 InfoNCE 是互信息的下界

van den Oord 等人证明[^1]，InfoNCE 损失与互信息存在如下关系：

I(zt;ct)≥LInfoNCE+log⁡KI(z_t; c_t) \geq \mathcal{L}_{\text{InfoNCE}} + \log K I(zt​;ct​)≥LInfoNCE​+logK

即 InfoNCE 是互信息的对数似然下界（up to 一个常数 log⁡K\log KlogK，其中 KKK 是负样本数）。最小化 InfoNCE 损失，等价于最大化查询与正样本之间的互信息估计。

这解释了对比学习为什么有效：当我们推动模型区分正负样本时，实际上是在让表示捕获"什么让正样本成为正样本"的本质信息——这些信息对预测未来、识别相似实例最有价值。

3.3 负采样的意义

直接计算互信息需要知道 partition function p(x)p(x)p(x)，这在高维空间中是 intractable 的。InfoNCE 通过负采样绕过这一障碍：用一组随机负样本近似分母中的求和。负样本数 KKK 越大，近似越准确，但计算成本也越高。

几点实践建议：

  • 数值稳定性：实际代码中通常在 logits 减去最大值，防止指数溢出
  • 负样本数：常取 4096（MoCo）或 batch 内所有其他样本（SimCLR）
  • batch 大小：通常越大越好，因为更多负样本提供更强的对比信号
  • 温度调度：有些工作前期用高温、后期用低温来平衡探索与收敛

5. 总结

InfoNCE 损失函数的核心逻辑可以归结为三点：

  1. 正样本拉近：让查询与正样本的相似度尽可能高
  2. 负样本推远：通过对比，让正样本在众多负样本中脱颖而出
  3. 隐式互信息最大化：最小化 InfoNCE 等价于最大化查询与正样本之间的互信息

温度系数 τ\tauτ 控制了对比的粒度，负样本数 KKK 决定了近似的精度——这两个超参是实际使用中最重要的调参方向。

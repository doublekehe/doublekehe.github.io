从似然函数到损失函数

发表于2026-06-03|更新于2026-06-03|机器学习损失函数

当我们训练一个神经网络时，几乎理所当然地使用 Cross-Entropy（交叉熵）作为损失函数。但你有没有想过：为什么是它？它是从哪里来的？很多人会直接回答"因为效果好"，但真正的原因是——它是从似然函数自然推导出来的。本文用一枚硬币，把这个逻辑讲清楚。

1. 什么是似然函数

在讲似然函数之前，先回忆一下我们熟悉的概率。

假设有一枚硬币，正面朝上的概率是 ppp（0≤p≤10 \leq p \leq 10≤p≤1）。我们抛 nnn 次，定义 XXX 为正面出现的次数。学过概率论的同学应该知道：

P(X=k)=(nk)pk(1−p)n−kP(X = k) = \binom{n}{k} p^k (1-p)^{n-k} P(X=k)=(kn​)pk(1−p)n−k

这是标准的二项分布。给定参数 ppp，我们可以计算观察到 kkk 次正面的概率是多少。

注意这里的逻辑方向：已知模型参数 ppp，求数据 XXX 出现的概率。这就是概率的核心含义。

现在反过来想这个问题：我们实际抛了一枚硬币 10 次，观察到 7 次正面。请问这枚硬币正面朝上的概率 ppp 最可能是多少？

这个问题在概率论里没有直接答案——因为概率论关心的是"给定参数看结果"，而我们要做的是"看到结果后反推参数"。这个"反推"的动作，就是似然（Likelihood）。

似然函数就是这样定义的：给定观测到的数据 X=kX = kX=k，把它看成关于参数 ppp 的函数

L(p)=(nk)pk(1−p)n−kL(p) = \binom{n}{k} p^k (1-p)^{n-k} L(p)=(kn​)pk(1−p)n−k

注意它和概率表达式的形式完全一样，但语义不同：概率是在问"如果 ppp 是这个值，观察到 kkk 次正面的可能性有多大"；似然是在问"给定观察到 kkk 次正面，哪个 ppp 最能解释这个观测"。

2. 抛硬币——从概率到似然

让我们把上面的对比用具体数字走一遍，感受更深刻。

概率视角

假设我们有一枚均匀硬币（p=0.5p = 0.5p=0.5），抛 10 次，问：恰好出现 7 次正面的概率是多少？

P(X=7)=(107)×0.57×0.53=120×1210≈0.117P(X=7) = \binom{10}{7} \times 0.5^7 \times 0.5^3 = 120 \times \frac{1}{2^{10}} \approx 0.117 P(X=7)=(710​)×0.57×0.53=120×2101​≈0.117

这是概率论的标准问题。已知 p=0.5p = 0.5p=0.5，算 X=7X = 7X=7 的概率。

似然视角

反过来：实际抛了 10 次，观察到 7 次正面。现在问你：这枚硬币的 ppp 最可能是多少？

这就是似然函数要回答的问题。把 k=7,n=10k = 7, n = 10k=7,n=10 代入：

L(p)=(107)⋅p7⋅(1−p)3L(p) = \binom{10}{7} \cdot p^7 \cdot (1-p)^3 L(p)=(710​)⋅p7⋅(1−p)3

注意：(107)\binom{10}{7}(710​) 是常数，不影响 ppp 的选择。我们只需最大化

L~(p)=p7(1−p)3\tilde{L}(p) = p^7 (1-p)^3 L~(p)=p7(1−p)3

一个直观的方法是画出这个函数的图像——它在 p=0.7p = 0.7p=0.7 处取到峰值（可以用求导验证）。也就是说，给定观测到 7 次正面，最合理的 ppp 估计就是 0.7。

这就是**极大似然估计（Maximum Likelihood Estimation, MLE）**的核心思想：选择使观测结果出现概率最大的参数值作为估计。

对数似然

直接最大化 L(p)L(p)L(p) 乘积形式在数学上不太方便，取对数变成求和：

log⁡L(p)=log⁡(nk)+klog⁡p+(n−k)log⁡(1−p)\log L(p) = \log \binom{n}{k} + k \log p + (n-k) \log (1-p) logL(p)=log(kn​)+klogp+(n−k)log(1−p)

去掉常数项，最大化似然等价于最大化

klog⁡p+(n−k)log⁡(1−p)k \log p + (n-k) \log (1-p) klogp+(n−k)log(1−p)

这个对数形式非常关键——后续所有损失函数的推导，都是从这一步开始的。

3. 极大似然估计——机器学习的统一框架

硬币的例子可以自然推广到任意机器学习模型。

模型即"参数化的硬币"

想想神经网络的分类模型：输入一张图片，输出各类别的概率分布。模型的参数 θ\thetaθ（权重和偏置）决定了从输入到输出的映射。

如果模型说"这张图片是猫的概率是 0.9，是狗的概率是 0.1"，这就好比硬币说"正面的概率是 0.9"。

所以我们可以把模型的输出理解为对真实分布的参数化近似。训练模型，就是找到一组参数 θ\thetaθ，使得"给定这组参数，观测到当前这批训练数据"这件事的概率最大。

这就是极大似然估计在机器学习中的意义：模型训练 = 寻找使观测数据概率最大的参数。

MSE 损失也是极大似然

你可能没想到，回归任务中常见的 MSE（均方误差）损失，其实也可以从极大似然的框架里自然推出。

假设真实标签 yyy 和预测值 y^\hat{y}y^​ 之间存在关系：

y=y^+ϵ,ϵ∼N(0,σ2)y = \hat{y} + \epsilon, \quad \epsilon \sim \mathcal{N}(0, \sigma^2) y=y^​+ϵ,ϵ∼N(0,σ2)

也就是说误差服从均值为 0、方差为 σ2\sigma^2σ2 的高斯分布。那么 yyy 的概率密度为：

P(y∣y^)=12πσexp⁡(−(y−y^)22σ2)P(y|\hat{y}) = \frac{1}{\sqrt{2\pi}\sigma} \exp\left(-\frac{(y-\hat{y})^2}{2\sigma^2}\right) P(y∣y^​)=2π​σ1​exp(−2σ2(y−y^​)2​)

对于整个数据集，独立同分布假设下，总似然为各样本似然的乘积。取对数后化简，最后剩下的恰好是 MSE 损失：

−log⁡P(y∣y^)∝12(y−y^)2-\log P(y|\hat{y}) \propto \frac{1}{2}(y - \hat{y})^2 −logP(y∣y^​)∝21​(y−y^​)2

所以 MSE 不是凭空设计的，它在高斯噪声假设下就是负对数似然的必然结果。

4. 从似然函数到损失函数

现在到了本文最核心的部分：BCE 和 Softmax CE 这两个最常见的分类损失函数，是如何从似然函数一步步推导出来的。

Binary Cross-Entropy（BCE）—— 二分类

二分类问题是伯努利分布的直接应用。

设真实标签 y∈{0,1}y \in \{0, 1\}y∈{0,1}，模型预测为 y^=P(y=1∣x)\hat{y} = P(y=1|x)y^​=P(y=1∣x)。则单样本的似然函数为：

L(y^)=y^y⋅(1−y^)1−yL(\hat{y}) = \hat{y}^y \cdot (1-\hat{y})^{1-y} L(y^​)=y^​y⋅(1−y^​)1−y

取对数得到对数似然：

log⁡L=ylog⁡y^+(1−y)log⁡(1−y^)\log L = y \log \hat{y} + (1-y) \log (1-\hat{y}) logL=ylogy^​+(1−y)log(1−y^​)

在机器学习中，我们通常最小化负对数似然（Negative Log-Likelihood, NLL）：

BCE Loss=−log⁡L=−[ylog⁡y^+(1−y)log⁡(1−y^)]\text{BCE Loss} = -\log L = -[y \log \hat{y} + (1-y) \log (1-\hat{y})] BCE Loss=−logL=−[ylogy^​+(1−y)log(1−y^​)]

这就是 Binary Cross-Entropy。它不是凭空发明的——它就是伯努利分布的负对数似然。

Softmax Cross-Entropy —— 多分类

多分类是二分类的自然推广。

设类别数为 CCC，真实标签为 one-hot 向量 y=(y1,y2,…,yC)\mathbf{y} = (y_1, y_2, \ldots, y_C)y=(y1​,y2​,…,yC​)，模型输出的未归一化分数为 z=(z1,z2,…,zC)\mathbf{z} = (z_1, z_2, \ldots, z_C)z=(z1​,z2​,…,zC​)，通过 Softmax 归一化：

y^i=P(yi=1∣z)=exp⁡(zi)∑j=1Cexp⁡(zj)\hat{y}_i = P(y_i=1|\mathbf{z}) = \frac{\exp(z_i)}{\sum_{j=1}^{C} \exp(z_j)} y^​i​=P(yi​=1∣z)=∑j=1C​exp(zj​)exp(zi​)​

单样本的似然为：

L(z)=∏i=1Cy^iyiL(\mathbf{z}) = \prod_{i=1}^{C} \hat{y}_i^{y_i} L(z)=i=1∏C​y^​iyi​​

由于 y\mathbf{y}y 是 one-hot 的，实际上只有真实类别 c∗c^*c∗ 的 y^c∗\hat{y}_{c^*}y^​c∗​ 起作用，其他都是 0 次方等于 1。所以对数似然简化为：

log⁡L=log⁡y^c∗=log⁡exp⁡(zc∗)∑jexp⁡(zj)=zc∗−log⁡∑jexp⁡(zj)\log L = \log \hat{y}_{c^*} = \log \frac{\exp(z_{c^*})}{\sum_j \exp(z_j)} = z_{c^*} - \log \sum_j \exp(z_j) logL=logy^​c∗​=log∑j​exp(zj​)exp(zc∗​)​=zc∗​−logj∑​exp(zj​)

负对数似然即为 Softmax Cross-Entropy Loss：

CE Loss=−zc∗+log⁡∑j=1Cexp⁡(zj)\text{CE Loss} = -z_{c^*} + \log \sum_{j=1}^{C} \exp(z_j) CE Loss=−zc∗​+logj=1∑C​exp(zj​)

这就是你在 PyTorch / TensorFlow 里用 CrossEntropyLoss 时实际计算的公式。它的来源不是别的，就是多项分布的负对数似然。

5. 总结

回顾全文，我们走过了这样一条路：

步骤 内容
1. 概率 已知参数 ppp，问数据的分布 P(X∣p)P(X|p)P(X∣p)
2. 似然 已知数据 XXX，反推最合理的参数 L(p∣X)L(p|X)L(p∣X)
3. 对数似然 取对数把乘积变求和，方便优化
4. 负对数似然 最小化 NLL 等价于最大化似然
5. BCE / CE NLL 在伯努利 / 多项分布下的具体形式

所以，当你下次使用 CrossEntropyLoss 或 BCELoss 时，你用的不是别人随便设计的一个公式——它是从似然原理一路自然推导出来的结论。理解了这个来龙去脉，你不仅知道这些损失函数怎么用，还知道它们为什么长成这样，以及在什么假设下它们是最优的选择。

这就是似然函数作为"桥梁"的意义：一端是概率论的严格数学框架，另一端是神经网络训练的实际损失函数。理解了这座桥，很多看似复杂的公式和概念，都会变得清晰而自然。

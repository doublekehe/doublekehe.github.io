但这是错误的做法。原因是：

  • [[0] * m] 先创建了一个只有一行的二维列表；
  • * n 操作并不会复制内部列表的内容，而是复制引用；
  • 所以最终得到的 matrix 中，所有行都指向同一个内层列表对象。

为什么要区分行赋值和列赋值

  • 行赋值可以直接替换整行对象，因为外层列表存储的是行列表的引用；
  • 列赋值不能直接替换列对象，因为列不是一个独立的对象，而是每一行中的同一位置元素的集合。

因此列赋值只能通过遍历每一行来实现。

结论

  • Python 二维 list 本质上是“列表的列表”。
  • 推荐使用 [[0] * m for _ in range(n)] 进行初始化。
  • 绝不要使用 [[0] * m] * n，因为它会导致每一行指向同一个列表对象。
  • 赋值时要区分“行赋值”和“列赋值”：
    • 行赋值可以直接替换行对象；
    • 列赋值需要遍历每一行逐个修改对应列。
